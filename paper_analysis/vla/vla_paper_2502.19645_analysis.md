## 4. [2502.19645] Fine-tuning vision-language-action models: Optimizing speed and success

**一句话总结**: 提出优化的VLA微调方法(OFT)，通过并行解码和L1回归将OpenVLA的推理速度提升26倍，任务成功率从76.5%提升至97.1%。

**背景与动机**:
当前的VLA (Vision-Language-Action，视觉-语言-动作模型) 虽然在多样化任务上表现出色，但在部署到新机器人平台时面临两大核心问题：**推理速度慢**(3-5 Hz)无法满足高频控制需求(25-50 Hz)，以及**双臂操作任务成功率低**。现有的自回归VLA微调方法（如OpenVLA原始方案）需要逐token生成动作，导致无法实现动作分块 (action chunking)——因为生成K步动作需要K倍的延迟时间。这使得VLA难以应用于需要快速响应的灵巧操作任务。

**核心创新**:
这篇论文通过系统性实验研究了VLA微调的关键设计选择，提出了OFT (Optimized Fine-Tuning) 微调方案，包含三个核心技术点：

1. **并行解码 (Parallel Decoding) + 动作分块 (Action Chunking)**
   - **做了什么**: 改变模型的注意力机制，让模型可以一次性生成多个时间步的动作，而不是逐个生成
   - **怎么做的**: 将OpenVLA的因果注意力掩码 (causal attention mask) 替换为双向注意力 (bidirectional attention)，让解码器接收空的动作嵌入 (empty action embeddings) 作为输入，然后在单次前向传播中直接映射到所有动作输出。对于K个时间步的动作块，只需插入K×D个空嵌入（D是动作维度），模型就能一次性输出K步动作

2. **连续动作表示 (Continuous Action Representation)**
   - **做了什么**: 用连续的动作值替代原来的离散化token表示，提高动作预测精度
   - **怎么做的**: 移除原来的256-bin离散化方案，在解码器后面接一个MLP动作头 (action head)，直接将隐藏状态映射到归一化的连续动作值[-1, +1]。这样可以保留更细粒度的动作细节

3. **L1回归目标 (L1 Regression Objective)**
   - **做了什么**: 用简单的L1损失替代复杂的扩散模型 (diffusion model) 训练目标，保持性能同时大幅提升训练和推理效率
   - **怎么做的**: 训练时最小化预测动作和真实动作之间的平均L1距离。相比扩散模型需要50步去噪，L1回归只需单次前向传播就能生成动作

**技术方案**:

### 整体架构
OpenVLA-OFT保留原OpenVLA的视觉-语言编码器（Prismatic VLM基座），但重新设计了动作生成部分：
- **输入处理**: 支持多视角相机图像（每个视角提取256个patch embeddings）+ 机器人状态（关节角度等）+ 语言指令，所有输入通过投影网络映射到统一的语言嵌入空间
- **解码器**: 使用双向注意力的Transformer解码器，一次性处理K×D个动作查询
- **输出头**: MLP动作头直接输出连续动作值

### 关键技术细节

**1. 并行解码的实现原理**
- 原OpenVLA需要7次串行前向传播（3维位置+3维方向+1维夹爪）
- 并行解码将7次压缩为1次，通过在解码器输入中插入7个空嵌入，利用双向注意力让模型同时"看到"所有需要生成的动作位置
- 结合动作分块时，对于K=8的分块，只需1次前向传播就生成56维动作（8步×7维），吞吐量提升26倍

**2. 为何连续动作优于离散动作**
- 离散化损失精度：原方案将每个动作维度均匀离散化为256个bins，归一化后的动作空间[-1, +1]被切分为约0.008的步长
- 连续表示直接输出浮点数值，保留了机器人控制所需的精细调整能力
- 实验显示连续动作版本在LIBERO基准上相比离散版本提升5%成功率

**3. L1回归 vs 扩散模型的权衡**
- 扩散模型理论上更具表达力（能建模多模态分布），但训练收敛慢（需要100-250K步 vs L1的50-150K步）
- 推理时扩散需要50步去噪（延迟1.9秒），L1只需1步（延迟0.07秒）
- 实验发现高容量的OpenVLA基座模型用简单L1就能有效建模多任务动作分布，两者成功率相当（95.3% vs 95.4%）

**4. FiLM语言增强（OFT+变体）**
- **动机**: 在多视角输入时（如ALOHA的3个相机），模型容易学到视觉输入中的虚假相关性而忽略语言指令
- **方法**: 使用特征级线性调制 (Feature-wise Linear Modulation)，将语言嵌入投影为缩放向量γ和平移向量β，对视觉特征进行仿射变换：F̂ = (1+γ) ⊙ F + β
- **关键实现细节**: 不是对每个patch embedding单独调制，而是让γ和β跨所有patches对相同的特征维度进行调制（类似CNN中的空间无关调制），在每个ViT block的自注意力层后、前馈层前应用

### 与现有方法的区别

| 方面 | 原OpenVLA | RDT-1B/π₀ (扩散VLA) | OpenVLA-OFT |
|------|-----------|---------------------|-------------|
| 解码方式 | 自回归（串行） | 并行（扩散去噪） | 并行（直接映射） |
| 动作表示 | 离散token | 连续值 | 连续值 |
| 训练目标 | 交叉熵（next-token） | 扩散去噪 | L1回归 |
| 推理步数 | D次前向传播 | 50步去噪 | 1次前向传播 |
| 吞吐量 (单臂) | 4.2 Hz | ~4 Hz | 109.7 Hz (26×) |
| 双臂吞吐量 | ~2 Hz | ~4 Hz | 71.4 Hz (35×) |

**技术优势**：
- 相比自回归VLA（如FAST），延迟从750ms降至0.07ms（单臂）或0.32ms（双臂）
- 相比扩散VLA，训练速度快2倍，推理速度快50倍（在保持相同性能前提下）
- 架构简单，不需要复杂的噪声调度或多步采样策略

**实验效果**:

### LIBERO仿真基准测试

**主要结果**:
- **成功率对比** (四个任务套件平均):
  - 原OpenVLA (自回归): 76.5%
  - OpenVLA + 并行解码&动作分块: 90.2% (+13.7%)
  - OpenVLA-OFT (完整方案): 95.3% (+18.8%)
  - OpenVLA-OFT (含额外输入): **97.1% (SOTA)** (+20.6%)
  
- **分任务表现** (含额外输入版本):
  - LIBERO-Spatial (不同空间布局): 97.6%
  - LIBERO-Object (不同物体): 98.4%
  - LIBERO-Goal (不同目标): 97.9%
  - LIBERO-Long (长时域任务): 94.5%

- **与其他方法对比**:
  - 超越π₀ (94.2%) 提升2.9%
  - 超越Diffusion Policy从零训练 (72.4%) 提升24.7%
  - 超越Octo微调 (75.1%) 提升22.0%

**推理效率提升** (A100 GPU，7维动作):
- 延迟: 240ms → 73ms (降低70%)
- 吞吐量: 4.2 Hz → 109.7 Hz (**26倍加速**)
- 加入额外输入后仍保持71.4 Hz (17倍加速)

### ALOHA真实双臂机器人测试

**任务设置**:
4个灵巧操作任务，测试可形变物体、长时域操作、工具使用和语言驱动控制：
1. 折叠短裤 (20个演示)
2. 折叠T恤 (30个演示)  
3. 用勺子舀指定食材到碗里 (45个演示，3种食材)
4. 把指定物品放进锅里 (300个演示，3种物体)

**性能对比** (任务完成度评分):
- **从零训练方法**:
  - ACT: 平均分最低（动作精度差）
  - Diffusion Policy: 在折叠和舀食材任务上与RDT-1B相当，但在大数据集任务上泛化性差
  
- **微调VLA方法**:
  - RDT-1B: 语言跟随能力好，但整体任务执行不如OpenVLA-OFT+
  - π₀: 平均表现仅次于OpenVLA-OFT+
  - **OpenVLA-OFT+**: 在所有4个任务上获得最高平均分，比最强基线高15%

**语言跟随能力**:
- 在"舀X到碗里"任务中，OpenVLA-OFT+对3种食材的选择准确率均达到最高
- 在"把X放进锅里"任务中，对3种物体的识别和操作成功率最高
- 证明FiLM有效增强了语言理解能力

**控制频率**:
- 原OpenVLA无法在ALOHA上部署（<2 Hz，远低于25 Hz需求）
- OpenVLA-OFT+在K=25动作分块下达到43倍加速，满足25 Hz实时控制要求

### 消融实验验证

**动作分块的重要性**:
- 不仅提升速度，还能提升性能（尤其在长时域任务LIBERO-Long上从53.7%→86.5%）
- 原因：减少累积误差，捕获时序依赖关系

**连续vs离散动作**:
- 连续动作版本比离散版本高5%成功率
- 说明动作精度对机器人控制的重要性

**L1 vs 扩散**:
- 性能相当（95.3% vs 95.4%），但L1训练快2倍、推理快50倍
- 扩散的多步采样优势在高容量模型上不明显

**预训练的价值**:
- 移除OpenVLA预训练后直接从VLM微调，LIBERO平均成功率下降5.2%
- 证明机器人预训练仍然有效，即使微调时输入输出格式发生变化

**个人点评**:
这篇工作的最大价值在于**系统化地拆解了VLA微调的设计空间**，通过控制变量实验证明了"并行解码+连续动作+L1回归"这个简单组合能超越复杂的扩散方法。亮点是**用奥卡姆剃刀原则解决工程问题**——在相同性能下选择最简单的方案，使得VLA真正可部署。局限在于FiLM增强是针对多视角场景的临时方案，未来可能需要更根本的语言理解架构改进。该工作为VLA实用化开辟了清晰路径，预期将成为后续VLA微调的标准范式。
