## 7. [2501.15830] Spatialvla: Exploring spatial representations for visual-language-action model

### 1. Motivation（研究动机）

**问题背景**：
当前的 Vision-Language-Action (VLA) 模型在构建通用机器人策略方面取得了显著进展，它们通过在大规模跨机器人数据集上微调预训练的 Vision-Language Models (VLMs) 来实现多任务泛化。然而，现有 VLA 模型主要局限于 2D 观察输入，缺乏对 3D 物理世界的精确感知和理解能力。人类在操控任务中能够本能地构建丰富的、结构化的空间心理表征，轻松地将物体对齐到一个规范的、直观的工作空间中，而这种 3D 空间智能在现有 VLA 模型中严重缺失。

**为什么重要**：
空间理解是机器人操控的关键能力。在发展具有 3D 空间智能的通用机器人策略时，需要解决两个核心挑战：（1）不同机器人实体的观察空间未对齐——由于相机传感器安装位置不同（如手腕视角或第三人称视角），导致 3D 观察空间未经校准；（2）不同机器人的动作运动特性差异显著——由于自由度、运动控制器、工作空间配置和任务复杂度不同，难以学习可泛化的空间动作表征。缺乏空间对齐的观察和动作表征，严重限制了跨实体控制和在通用 3D 物理世界中的适应能力。

**现有方法的不足**：
尽管已有一些尝试探索跨异构机器人的通用策略学习（如 Octo、HPT、RT-X 系列），但在通用策略的 3D 空间理解能力方面的进展明显滞后。现有方法如 RT-2、OpenVLA 等主要采用基于 VLM 的自回归范式，将连续动作离散化为固定的 256 个 bins，这种线性离散化方式未考虑动作空间的分布特性，难以捕捉精细的空间运动特征。此外，这些模型忽略了 3D 空间信息的显式建模，依赖纯 2D 语义特征进行决策，在处理涉及空间布局变化、高度差异或精确定位的任务时表现不佳。核心 gap 在于缺乏统一的、机器人无关的 3D 空间表征框架，无法有效桥接异构机器人的观察输入和动作输出。

### 2. Contribution（核心贡献）

本文的主要贡献包括：

1. **提出 SpatialVLA 模型架构**：首次系统性地探索 VLA 模型的空间表征问题，通过 Ego3D Position Encoding 和 Adaptive Action Grids 两大创新设计，赋予 VLA 模型 3D 空间智能，实现机器人无关的观察和动作空间统一表征。

2. **Ego3D Position Encoding**：设计了以自我为中心的 3D 位置编码方法，通过深度估计和相机内参反投影构建自我中心坐标系，将 3D 空间信息与 2D 语义特征融合。该方法无需机器人-相机外参校准，具有通用性，可适用于各种机器人实体和相机配置。

3. **Adaptive Action Grids**：提出自适应动作网格方法，通过对整个数据集的动作分布进行高斯拟合，自适应地将连续动作空间离散化为高分辨率的空间网格（8194 个 tokens，远超传统 256 bins），学习空间动作 tokens 以对齐跨机器人动作与 3D 物理世界的空间结构。关键创新在于通过三线性插值实现预训练动作网格到新机器人设置的快速迁移（Spatial Embedding Adaption）。

4. **卓越的实验验证**：在 1.1M 真实机器人演示数据上进行预训练，在 7 种机器人学习场景、24 个真实机器人任务和 3 个仿真环境中进行了广泛评估。在 SimplerEnv 基准上，零样本泛化性能显著超越 RT-2-X（71.9% vs 60.7% Visual Matching）和 OpenVLA（71.9% vs 27.7%），同时推理速度达到 20Hz，是 OpenVLA 的 3 倍以上。在真实 WidowX 和 Franka 机器人上展示了出色的空间理解能力、指令遵循能力和跨实体适应能力。

### 3. Method（技术方法）

**3.1 整体框架**

SpatialVLA 基于 PaliGemma2 视觉-语言模型开发，采用自回归范式预测机器人动作。模型接收图像观察 $o_t = \{I_1^t, ..., I_n^t\}$ 和自然语言任务指令 $L$ 作为输入，学习映射函数 $\tau(\cdot)$ 生成动作序列 $A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]$。核心创新在于通过 Ego3D Position Encoding 增强输入观察的 3D 空间感知能力，通过 Adaptive Action Grids 实现动作的空间化表征和预测。

**3.2 核心技术**

**(1) Ego3D Position Encoding（自我中心 3D 位置编码）**

- **作用**：将 3D 空间信息注入 2D 视觉特征，使模型能够理解场景的空间结构，消除不同机器人相机配置带来的观察空间差异。

- **设计**：
  1. 使用 ZoeDepth 估计深度图 $D$，通过反投影 $\pi^{-1}$ 和相机内参计算每个像素在自我中心坐标系中的 3D 位置 $p = \{x, y, z\}$
  2. SigLIP 视觉编码器提取 2D 语义特征 $X \in \mathbb{R}^{d \times h \times w}$
  3. 计算特征对应的 3D 位置 $P \in \mathbb{R}^{3 \times h \times w}$
  4. 通过正弦函数 $\gamma(\cdot)$ 和可学习 MLP 将 3D 位置编码为嵌入 $P' \in \mathbb{R}^{d \times h \times w}$
  5. 融合空间和语义信息：$O_{3d} = X + P' = X + \text{MLP}(\gamma(P))$

- **为什么这样设计**：自我中心坐标系避免了对特定机器人-相机外参的依赖，实现了机器人无关的 3D 表征。使用深度估计而非点云作为输入，提高了实用性和效率。正弦位置编码结合 MLP 能够学习灵活的空间特征表达，适应不同任务和场景的空间关系。

**(2) Adaptive Action Grids（自适应动作网格）**

- **作用**：将异构机器人的连续动作空间统一为离散的空间网格表征，使 VLM 能够通过自回归方式预测动作 tokens，同时捕捉精细的空间运动特性。

- **设计**：
  1. **动作分解**：将 7 维单臂动作分解为 $a = \{a_{\text{trans}}, a_{\text{rot}}, a_{\text{grip}}\}$，其中平移 $a_{\text{trans}} = \{x, y, z\}$、旋转 $a_{\text{rot}} = \{\text{roll, pitch, yaw}\}$、夹爪 $a_{\text{grip}}$ 为二元状态
  2. **坐标变换**：将笛卡尔坐标的平移转换为极坐标 $(\phi, \theta, r)$，解耦运动方向和距离
  3. **分布拟合**：对整个数据集的每个动作变量进行归一化到 $[-1, 1]$，拟合高斯分布 $\mathcal{N}(\mu_a, \Sigma_a)$
  4. **网格分割**：根据概率密度函数进行自适应分割，使每个区间的概率相等（$1/M$）：
     $$a_2, ..., a_M = \arg\min_{a_2,...,a_M} \left| \int_{a_i}^{a_{i+1}} f(x)dx - 1/M \right|, \quad i=1,...,M$$
     对运动方向 $\{\phi, \theta\}$ 分配更多网格以捕捉精细方向，最终得到 $M_{\text{trans}} = M_\phi \cdot M_\theta \cdot M_r$ 个平移网格和 $M_{\text{rot}} = M_{\text{roll}} \cdot M_{\text{pitch}} \cdot M_{\text{yaw}}$ 个旋转网格
  5. **Token 嵌入**：为每个网格学习嵌入 $E_a = \{E_{\text{trans}}, E_{\text{rot}}, E_{\text{grip}}\}$，总 token 数 $V = M_{\text{trans}} + M_{\text{rot}} + 2$（本文中 $V = 8194$）

- **为什么这样设计**：线性离散化（如 RT-2/OpenVLA 的 256 bins）忽略了动作分布的不均匀性，导致高频动作区域分辨率不足。自适应网格根据真实动作分布分配更多网格到常见运动模式，提高了表征效率。极坐标分解使方向和距离独立建模，更符合人类空间认知。高分辨率网格（8194 tokens）虽然增加了词汇量，但通过预测 3 个 tokens（平移、旋转、夹爪各 1 个）而非 7 个，实际推理速度更快（20Hz vs OpenVLA 的 5-6Hz）。

**(3) Spatial Embedding Adaption（空间嵌入适应）**

- **作用**：在后训练阶段快速适应新机器人设置，避免从头学习动作表征。

- **设计**：
  1. 对新机器人数据集拟合新高斯分布 $\mathcal{N}(\mu_{\text{new}}, \Sigma_{\text{new}})$
  2. 生成新动作网格 $G_{\text{new}}$
  3. 通过三线性插值初始化新 token 嵌入 $E_{a_{\text{new}}}$：
     $$e_i^{a_{\text{new}}} = \sum_{j=1}^K w_j e_j^a$$
     其中 $w_j$ 是根据新网格中心与预训练网格中心的距离计算的归一化权重
  4. 在新数据集上微调模型参数和新嵌入

- **为什么这样设计**：直接随机初始化新动作 tokens 会丢失预训练学到的空间动作知识。三线性插值利用预训练网格的空间邻域关系，保留了动作空间的连续性和平滑性。实验表明（表 V），该方法在小规模数据集上比全参数微调提升 4.6%-5.4%，在大规模数据集上也有显著改进，证明了空间知识的有效迁移。

**3.3 算法流程**

**训练阶段**：
1. 从数据集中随机采样批次 $[(o_{t_1}, A_{t_1}, L_i), ..., (o_{t_B}, A_{t_B}, L_j)]$
2. 对每个样本：
   - 计算 Ego3D 表征 $O_{3d}$ 
   - 编码任务指令 $L$
   - 将连续动作 $a_t$ 映射到动作网格 tokens $\tilde{a}_t$
3. 通过自回归预测 tokens，优化交叉熵损失：$\mathcal{L}(\theta) = \mathbb{E}_{p(A_t|o_t)} \mathcal{L}(a_t, \tilde{a}_t)$
4. 冻结文本 token 嵌入以保留通用知识，优化视觉编码器、LLM 主干、MLP 和动作 token 嵌入

**推理阶段**：
1. 输入当前观察 $o_t$ 和任务指令 $L$
2. 生成 Ego3D 表征 $O_{3d}$
3. 自回归预测 3 个动作 tokens（平移、旋转、夹爪各 1 个）
4. 解 token 化为连续动作 $a_t$，通过查询网格中心并插值
5. 预测 $T=4$ 步动作 chunk，执行集成动作后预测下一个 chunk

**3.4 与现有方法的区别**

| 方面 | RT-2 / OpenVLA | Octo | π0 | SpatialVLA（本文） |
|------|---------------|------|----|--------------------|
| **观察表征** | 纯 2D 语义特征 | 2D 特征 + Transformer | 2D 特征 + 流匹配 | 2D 语义 + 3D 空间（Ego3D） |
| **动作表征** | 线性 256 bins（7 tokens/动作） | 连续动作 | 连续动作（Flow Matching） | 自适应空间网格（3 tokens/动作） |
| **动作预测** | 自回归分类 | 扩散模型 | 流匹配 | 自回归空间 token 分类 |
| **跨实体适应** | 全参数微调 / LoRA | Embodiment stem + 微调 | 专家模块微调 | 空间嵌入适应 + LoRA |
| **推理速度** | OpenVLA ~5-6Hz | Octo ~10Hz | π0 ~6.5Hz | SpatialVLA ~20Hz |
| **预训练数据** | OXE（RT-2-X：1.4M，OpenVLA：1.0M） | OXE 0.8M | 未公开 | OXE + RH20T 1.1M |

**关键差异点**：
1. **空间感知能力**：SpatialVLA 是首个显式建模 3D 空间信息的 VLA 模型，通过 Ego3D 编码实现机器人无关的空间表征，而现有方法依赖 VLM 从 2D 图像隐式推断空间关系，在空间布局变化任务中表现不佳
2. **动作表征效率**：自适应网格根据数据分布自适应分配分辨率，比线性离散化更高效；通过极坐标分解和维度压缩，仅需 3 个 tokens 表达完整动作，而 RT-2/OpenVLA 需要 7 个，推理速度提升 3-4 倍
3. **迁移学习策略**：空间嵌入适应通过网格重新离散化和三线性插值，实现预训练知识的快速迁移，而传统 LoRA 微调未利用动作空间的几何结构

### 4. Experiment（实验验证）

**4.1 实验设置**

- **数据集**：
  - 预训练：OXE（Open X-Embodiment）子集 + RH20T，共 1.1M 真实机器人演示，覆盖 22 个机器人实体和 21 个机构
  - 微调：Google Fractal、BridgeData V2、LIBERO（4 个任务套件，每套 10 任务 × 50 演示）、自建 Franka 数据集（13 任务）

- **基线方法**：
  - 专家策略：RT-1、RT-2-X（闭源 55B 参数）、Diffusion Policy
  - 跨实体通用策略：RT-1-X、Octo-Base/Small、HPT、OpenVLA（7B 参数）、RoboVLM、TraceVLA
  - 最新 SOTA：π0（BF16 uniform 版本）

- **评估指标**：
  - SimplerEnv：Visual Matching（视觉匹配成功率）、Variant Aggregation（变体聚合成功率）
  - 真实机器人：任务成功率（Success Rate）
  - LIBERO：平均成功率（SR）和标准误差，三个随机种子 × 500 次试验

- **实验环境**：
  - 仿真：SimplerEnv（Google Robot + WidowX 设置）、LIBERO（4 套任务）
  - 真实机器人：WidowX（7 任务套件 × 11 次试验 = 77 rollouts）、Franka（13 任务，单任务 / 指令遵循 / 多任务）

**4.4 分析与讨论**

#### **4.4.1 空间理解能力验证（图 7）**

设计了 4 个专门测试空间理解的任务：

1. **WidowX 零样本任务 #1**："Place plush toy **closest to** robot on car"（空间方位指令）
   - **SpatialVLA**：73% 成功率
   - OpenVLA / Octo：< 50%
   - **分析**：需要理解"closest to"的相对位置关系，Ego3D 编码提供精确的机器人-物体距离信息

2. **WidowX 零样本任务 #2-4**：高度变化的放置任务（Put green cup on pink cloth，杯子高度增加）
   - **SpatialVLA**：保持高成功率（~80-90%）
   - Octo / Diffusion Policy：下降到 < 50%
   - **分析**：高度变化导致 2D 视觉外观剧烈变化，3D 空间表征使 SpatialVLA 对高度不变，2D 模型依赖像素位置失败

3. **Franka 微调任务 #1**："Place plush toy closest to robot on car"
   - **SpatialVLA**：73%
   - Octo：40%
   - **分析**：同 WidowX 任务 #1，跨机器人实体的空间理解泛化

4. **LIBERO-Spatial 套件**：10 个不同物体布局任务
   - **SpatialVLA**：88.2 ± 0.5%（第一）
   - OpenVLA：84.7 ± 0.9%（第二）
   - Octo：78.9 ± 1.0%（第四）
   - **分析**：布局变化是典型的空间理解挑战，SpatialVLA 显著领先

**总结**：所有涉及空间提示理解、高度变化、布局变化的任务中，SpatialVLA 均展现出显著优势（提升 10%-30%），证明 Ego3D 编码和空间动作网格有效赋予模型 3D 空间智能。

#### **4.4.2 推理速度优势**

| 模型 | 推理速度（Hz） | Tokens 数 / 动作 | 参数量 |
|------|---------------|------------------|-------|
| RT-2-X | 5.0 Hz | 7 | 55B |
| OpenVLA | 5.2 Hz | 7 | 7B |
| TraceVLA | 6.5 Hz | 7 | 7B |
| **SpatialVLA** | **20.1 Hz** | **3** | **3B** |

**分析**：
- **3 tokens vs 7 tokens**：通过极坐标分解和维度压缩（平移、旋转、夹爪各 1 个 token），减少自回归步数从 7 降至 3，理论加速 2.3 倍
- **高分辨率 vs 低分辨率**：虽然词汇量从 256 增至 8194，但 GPU 解码器对词汇量的计算复杂度影响远小于序列长度（$O(V)$ vs $O(T^2)$），实际推理瓶颈在 Transformer 自注意力
- **模型参数优化**：3B 参数量（vs OpenVLA 7B）减少计算量，PaliGemma2 主干更高效
- **实际测量**：单 NVIDIA RTX 4090 上，8.5GB 显存，20Hz 推理速度，满足实时机器人控制需求（>10Hz）

#### **4.4.3 Surprising 发现**

1. **预训练动作网格的迁移能力惊人**（表 V）：即使在分布差异大的小数据集（LIBERO）上，通过三线性插值初始化的新网格嵌入，相比随机初始化提升 4.6%-5.4%。说明空间动作 tokens 学到了可泛化的几何结构知识，而非特定数据集的统计规律。

2. **Ego3D 编码对变体聚合任务的巨大影响**（表 IV #8）：在"Put Eggplant"任务中，无 Ego3D 的模型成功率仅 37.5%，加上后跃升至 87.5%（+50%）。分析发现，该任务涉及从侧视角放置物体到篮子，2D 视角下物体遮挡严重，3D 信息至关重要。

3. **冻结语言嵌入的意外收益**（表 IV #9）：原本是为防止灾难性遗忘，意外发现冻结后指令遵循能力显著提升（+20%）。推测原因是机器人数据的语言标注质量不如 VLM 预训练的自然语言数据，微调语言嵌入反而降低语义理解能力。

4. **低分辨率网格导致"慢动作"问题**（表 IV #4）：1026 分辨率的模型在训练中可视化发现，预测的动作幅度持续偏小，机器人运动缓慢（需要更多步完成任务）。原因是低分辨率网格的量化误差大，模型为降低误差倾向于预测保守的小幅动作。

#### **4.4.4 局限性**

1. **长期任务性能不足**（LIBERO-Long：55.5%）：模型仅依赖当前帧和历史 tokens，缺少显式的长序列历史建模机制（如循环模块或 memory bank），在需要多步推理的长期任务中表现不佳。

2. **深度估计误差传播**：Ego3D 编码依赖 ZoeDepth 的单目深度估计，在反射表面、透明物体或极端光照下估计误差大，可能影响空间表征质量。未来可考虑融合双目立体视觉或结构光深度传感器。

3. **高斯分布假设的局限**：在极端机器人操作场景（如单轴运动、高度非对称工作空间）下，高斯分布拟合可能导致网格在某些轴上聚集，丢失其他轴的运动能力。数据集噪声也会扭曲网格分布。建议未来结合隐式分布建模（如 VAE 潜空间映射）和显式网格分割。

4. **扩散解码的缺失**：当前自回归范式预测单步动作，虽然快（20Hz），但无法像 π0、Diffusion Policy 那样一次解码多步连续动作，在需要平滑轨迹的任务（如绘画、倾倒）中可能不如扩散模型。未来可探索将空间网格表征与扩散解码器结合。

---

**总结**：SpatialVLA 通过创新的 Ego3D 位置编码和自适应动作网格设计，首次系统性地为 VLA 模型注入 3D 空间智能，在 7 种机器人场景、24 个真实任务和 3 个仿真环境中展现出全面领先的性能。零样本泛化能力超越所有开源模型，甚至媲美闭源 55B 参数的 RT-2-X；空间理解能力在涉及布局变化、高度差异和方位指令的任务中显著优于现有方法；推理速度达到 20Hz，是 OpenVLA 的 3-4 倍；空间嵌入适应方法为后训练提供了新范式。实验结果充分验证了空间表征在构建通用机器人策略中的关键作用。
