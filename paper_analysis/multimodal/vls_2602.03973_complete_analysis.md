## 3. [2602.03973 v1] VLS: Steering Pretrained Robot Policies via Vision-Language Models

### 1. Motivation(研究动机)

**问题背景**:
预训练的扩散或流匹配机器人策略(diffusion/flow-matching policies)在训练数据分布内表现优异,但在面对分布外(OOD)场景时表现脆弱——即使只是轻微改变物体位置、添加障碍物或调整支撑表面,策略就会失效。这些失败并非缺少运动技能(motor skills),而是因为模仿学习(imitation learning)将动作生成与训练时的空间配置和任务描述紧密耦合,导致测试时空间结构改变就无法泛化。现有的预训练VLA(Vision-Language-Action)模型尽管使用了VLM骨干网络,在联合观测-语言扰动下成功率仍急剧下降。

**为什么重要**:
这个问题阻碍了机器人策略的实际部署。重新训练或微调来应对每种OOD情况成本高昂且概念上不合理——因为所需的运动行为已经存在于训练数据中,只是无法在测试时选择性地适配。人类能轻松将"把杯子放在桌子中央"的技能迁移到"放在桌角"或"杂乱环境中",机器人却做不到,这限制了通用机器人策略的实用性。

**现有方法的不足**:
- **值函数引导方法**(如VGD, V-GPS):依赖额外学习的值函数,将策略偏向critic的偏好,破坏了基础策略的不变性
- **动力学模型引导**(如DynaGuide, Latent Policy Barrier):增加了对预测建模的依赖,对模型误差敏感,推理成本高
- **VLM-in-the-loop方法**(如ITPS, FOREWARN):提供的监督信号稀疏且离散,依赖候选动作序列的选择/拒绝,样本效率低且无法满足细粒度约束
- **在线改进方法**(如Policy Decorator, USR):需要在部署时进行在线学习,而非纯推理时适配

核心gap:**缺乏一个无需训练、仅在推理时就能将预训练策略适配到OOD场景的框架**,该框架需要提供稠密的、可微分的引导信号,同时保持基础策略参数冻结。

### 2. Contribution(核心贡献)

本文的主要贡献包括:

1. **VLS框架**:提出Vision-Language Steering(VLS),一个无需训练的推理时框架,通过steering冻结的生成式机器人策略的采样过程来适配OOD输入。核心思想是利用VLM的开放世界理解能力,为部分去噪的动作提案生成可微分的奖励函数,纠正去噪路径以适配测试时的空间和语义约束。

2. **三大技术组件**:
   - **OOD输入接地与奖励生成**:将高维观测-语言对压缩为3D关键点几何支架,利用VLM合成阶段感知的、程序化的、可微分的奖励函数
   - **推理时去噪引导**:结合梯度引导、RBF排斥力保持多样性、Feynman-Kac重采样三种机制,高效引导动作分布
   - **闭环执行控制与阶段切换**:自适应调节引导强度,采用Schmitt触发器机制实现鲁棒的多阶段任务切换

3. **通用性与实用性**:VLS同时适用于扩散策略和流匹配策略,无需修改策略参数,提供了一个即插即用的适配方案。相比需要额外学习辅助模型的方法,VLS直接利用现成的VLM和SAM,降低了部署门槛。

4. **实验验证**:
   - 在CALVIN上,VLS比基础策略提升7.4×(可移动物体)和9.6×(铰接部件),比先前最佳steering方法提升15-25个百分点,最高达到31%的绝对提升
   - 在LIBERO-PRO上,VLS在联合观测和语言扰动下比π0.5 LeRobot baseline提升13%
   - 真实世界Franka机器人部署中,VLS在分布内任务上提升19%,在最困难的物体替换OOD场景中baseline完全失败时VLS仍有40%成功率

### 3. Method(技术方法)

**3.1 整体框架**

VLS将OOD适配建模为推理时控制问题:给定冻结的基础策略π⋆和OOD输入对(o, l)_OOD,通过近似引导信号g ≜ ∇_{a^k} log p((o,l)_OOD | a^k)来steering采样过程,而不修改策略参数。核心是构造可微分的代理得分函数R(a^k, (o,l)_OOD),使其梯度能提供稠密的、动作空间级的引导。

**3.2 核心技术**

**(1) OOD输入接地(OOD Input Grounding)**

- **作用**:将高维的观测-语言对(o, l)_OOD压缩为低维的、任务相关的几何表示
- **设计**:
  1. 使用VLM识别任务相关的物体和区域
  2. 应用SAM获取物体掩码M
  3. 提取DINOv2稠密视觉特征Φ ∈ R^{H×W×d},用掩码过滤
  4. 结合深度信息将掩码像素反投影为3D点云
  5. 将点云聚类得到关键点集合P = {p_i}^n_{i=1},每个p_i ∈ R³锚定一个物理上有意义的空间约束
- **为什么这样设计**:点云聚类将高维RGB-D观测确定性地压缩为紧凑的几何支架,暴露下游可微分奖励评估所需的空间变量,避免直接在像素空间操作的高计算成本

**(2) 程序化奖励生成(Programmatic Reward Generation)**

- **作用**:为每个任务阶段合成可微分的奖励函数,评估动作提案满足空间约束的程度
- **设计**:
  1. VLM将任务分解为S个顺序阶段
  2. 为每个阶段s ∈ {1,...,S}生成奖励函数R_s(a^k, (o,l)_OOD) = f_VLM(a^k, P, s)
  3. R_s定义为阶段特定的势场,返回关于动作提案a^k可微分的标量值
  4. 约束VLM输出PyTorch可微分张量运算组成的程序化定义(距离、点积、软约束等)
- **为什么这样设计**:程序化奖励确保可微分性,梯度可回传;VLM作为非微分的离线组件,不参与梯度计算但提供跨模态推理能力;阶段分解使复杂任务的约束可分解为阶段特定的势场,便于精细控制

**(3) 多样性提案初始化(RBF排斥力)**

- **作用**:防止动作提案批次过早坍缩到基础策略的狭窄模式,保持全局覆盖
- **设计**:在早期去噪步骤引入基于成对距离的排斥梯度:
  ```
  g^k_RBF[i] = ∇_{a^k[i]} Σ_{j≠i} 1/(||a^k[i] - a^k[j]||² + ε)
  ```
  鼓励批次内的动作提案保持分离
- **为什么这样设计**:扩散/流匹配的批量采样容易收敛到局部模式,RBF排斥力在早期去噪阶段提供多样性,为后续奖励引导提供多样化候选,避免纯梯度引导导致的模式坍缩

**(4) 梯度引导精炼(Gradient-Based Refinement)**

- **作用**:将去噪轨迹偏向满足OOD约束的动作
- **设计**:
  1. 计算阶段特定的奖励梯度g_s = ∇_{a^k} R_s(a^k, (o,l)_OOD)
  2. 在每个去噪步k注入奖励梯度到噪声/速度预测:
     - 扩散策略:ε̂ = ε(a^k, (o,l)_OOD, k) - λ√(1-ᾱ_k)·g
     - 流匹配策略:v̂ = v(a^k, (o,l)_OOD, k) + λ·g
  3. 采用类似MCMC的随机精炼,每个去噪步多次内部更新
- **为什么这样设计**:稠密的轨迹级梯度提供精细的约束满足引导;多次内部更新提高在噪声梯度下的稳定性,减轻对局部梯度伪影的敏感性;梯度强度λ可调节,平衡基础策略先验与OOD约束

**(5) Feynman-Kac重采样(Gradient-Free Resampling)**

- **作用**:通过无梯度的重采样机制倾斜生成过程的转移核,使其向目标分布p(a|(o,l)_OOD)靠拢
- **设计**:
  1. 将动作提案批次解释为相互作用的粒子系统
  2. 为第i个粒子在去噪步k定义势函数:G^k_i = exp(R_s(a^k[i], (o,l)_OOD))
  3. 计算归一化权重:w^k_i = G^k_i / Σ_j G^k_j
  4. 周期性地根据权重进行多项式重采样
- **为什么这样设计**:梯度引导在局部区域有效,但可能陷入次优模式;FK重采样提供全局选择机制,让高奖励粒子复制而修剪违反约束的提案;两者结合实现了全局探索(重采样)与局部精炼(梯度)的互补,避免纯选择方法的样本低效

**(6) 自适应引导强度(Adaptive Guidance Strength)**

- **作用**:根据执行反馈动态调节引导强度λ
- **设计**:在固定阶段s内,为每个动作块t基于当前奖励R^t_s相对基线R^base_s计算:
  ```
  λ_t = λ_max · sigmoid(1 - R^t_s / R^base_s)
  ```
  偏离约束时增强引导,接近完成时减弱引导
- **为什么这样设计**:粗校正时需要强引导,精细操作时应让基础策略主导;自适应调度避免过度引导导致的不自然行为,同时确保在偏离时及时纠正

**(7) Schmitt触发器阶段切换(Schmitt-Trigger Stage Switching)**

- **作用**:鲁棒地判断何时在阶段间切换,避免在阶段边界振荡
- **设计**:定义高低阈值R_high和R_low,基于R^t_s的演化计算切换信号:
  ```
  Q_t = {
    Advance stage,    R^t_s > R_high
    Maintain stage,   R_low ≤ R^t_s ≤ R_high
    Reinforce stage,  R^t_s < R_low
  }
  ```
  触发切换时,查询VLM解释执行结果并选择下一阶段奖励R_{s+1}
- **为什么这样设计**:引入滞后避免过早切换和重复振荡;三状态机制(前进/维持/强化)提供了比简单阈值更鲁棒的状态转换;VLM参与切换决策利用其理解能力判断执行结果,而非仅依赖数值阈值

**3.3 算法流程**

完整的VLS算法(Algorithm 1)流程:
1. **条件接地**:P = {p_i} ← VLM(o_0, l)
2. **奖励生成**:{R_s(a, P)} ← f_VLM(o_0, l, P)
3. **采样初始化**:从N(0,I)采样B个动作提案{a^K[i]}
4. **去噪循环**(k = K → 0):
   - 应用RBF排斥梯度g^k_RBF
   - 计算奖励梯度g^k_reward
   - 执行M_MCMC次内部更新(扩散策略M=4,流匹配M=1)
   - 基于势函数exp(R_s)进行FK重采样
5. **闭环控制**:
   - 通过Eq.(10)自适应调整λ_t
   - 通过Eq.(11)更新阶段s
6. **输出**:返回a_{t:t+T}[0]

**3.4 与现有方法的区别**

| 维度 | VLS | DynaGuide | ITPS | VGD/V-GPS | Policy Decorator |
|------|-----|-----------|------|-----------|------------------|
| 训练需求 | 无需训练 | 需预训练动力学模型 | 无需训练 | 需学习值函数 | 需在线学习残差策略 |
| 引导信号 | VLM生成的可微奖励 | DINO特征距离启发式 | 离散人类信号 | 学习的值/Q函数 | 残差动作 |
| 信号密度 | 稠密轨迹级梯度 | 稠密但启发式 | 稀疏离散 | 稠密但学习偏好 | 稠密但需交互 |
| 基础策略不变性 | ✓ 完全冻结 | ✓ 冻结 | ✓ 冻结 | ✗ 被critic偏好改变 | ✗ 被残差策略修改 |
| OOD泛化机制 | VLM开放世界理解 | 固定特征空间 | 预定义函数集 | 固定值函数 | 在线适配 |
| 计算开销 | 中等(批采样+MCMC+FK) | 中等(动力学推演) | 低(选择/拒绝) | 低(值函数前向) | 中等(在线更新) |

**核心差异**:VLS是唯一结合了(1)无需训练、(2)VLM驱动的开放世界适配、(3)稠密可微引导、(4)基础策略完全不变性的方法。DynaGuide缺少语义理解,ITPS缺少稠密引导,VGD破坏策略不变性,Policy Decorator需要在线学习。

### 4. Experiment(实验验证)

**4.1 实验设置**

- **数据集**:
  - CALVIN:长期机器人操作基准,包含可移动物体(彩色立方体)和铰接部件(抽屉、开关、按钮、门)
  - LIBERO-PRO:基于LIBERO的OOD测试套件,包含4个子套件(Goal, Spatial, 10-Long, Object),每个子套件10个任务,每任务20集
  - 真实世界:Franka Emika机器人,2级任务复杂度
  
- **OOD扰动类型**:
  - **观测扰动**:(1)添加未见过的干扰物体 (2)改变物体属性 (3)改变物体位置/朝向
  - **语言扰动**:改变目标物体和目标行为
  - LIBERO-PRO测试位置扰动(o_OOD)和任务扰动(l_OOD)

- **基线方法**:
  - VLA模型:OpenVLA, π0, π0.5, π0.5 LeRobot(4个LIBERO-PRO排行榜顶级模型)
  - Steering方法:DynaGuide(DINO特征引导), ITPS(人类交互引导)
  - 消融变体:w/o gradient guidance, w/o FK resampling, w/o RBF diversity

- **评估指标**:成功率(Success Rate, SR%)、平均episode长度、推理时间

**4.2 主要结果**

**Q1: 推理时steering是否必要?**

LIBERO-PRO结果(Table I)显示现有VLA在OOD下急剧退化:
- OpenVLA和π0在所有任务和位置扰动下成功率均为0%
- π0.5性能略好但平均仅10.75%
- π0.5 LeRobot baseline平均23.69%
- **π0.5 LeRobot + VLS达到36.81%,提升13.12个百分点**

具体维度:
- 任务扰动下:VLS从23.13%提升到38.50%(+15.37%)
- 位置扰动下:VLS从24.25%提升到35.13%(+10.88%)
- Object套件任务扰动:VLS达到41.00%,相比baseline 10.50%提升3.9倍

**关键发现**:尽管VLA使用预训练VLM骨干,后训练在机器人数据上entangle了空间推理与训练上下文,降低了VLM在执行环境偏离训练流形时的泛化能力。推理时steering是预训练策略适配的必要条件。

**Q2: VLS相比现有steering方法的优势?**

CALVIN对比结果(Figure 3):

| 任务类型 | Base Policy | DynaGuide | ITPS | VLS | VLS提升倍数 |
|---------|------------|-----------|------|-----|------------|
| 可移动物体(平均) | 12.7% | ~65% | ~70% | **94%** | 7.4× vs base |
| 铰接部件(平均) | 9.0% | ~60% | ~72% | **87%** | 9.6× vs base |

VLS相比次优方法提升:
- 可移动物体任务:比DynaGuide/ITPS高15-25个百分点
- 铰接部件任务:比ITPS高15个百分点左右
- CALVIN整体最高达到**31%绝对提升**

分析:
- ITPS在铰接任务(固定目标状态)表现尚可,但在可移动物体任务(位置跨episode变化)失效
- DynaGuide的DINO特征启发式缺乏表达力,无法捕获任务特定空间需求
- VLS直接条件于当前观测-语言输入,能处理空间变异性和任务依赖约束

**4.3 消融实验**

**实验A - 去除梯度引导(w/o gradient guidance)**:
- 结果:成功率从88%暴跌到17.3%,episode长度从61步增加到178.6步
- 结论:**稠密的轨迹可微引导是VLS有效性的主要驱动因素**,去除后性能崩溃

**实验B - 去除FK重采样(w/o FKD)**:
- 结果:成功率略降到85.3%,episode长度增加到64.7步,推理时间从1189ms降到1170ms
- 结论:FK重采样对成功率影响较小但持续改善效率和稳定性,通过防止过早坍缩到次优模式提升样本效率

**实验C - 去除RBF多样性(w/o RBF)**:
- 结果:成功率降到86.0%,episode长度增加到69.3步
- 结论:RBF多样性初始化通过在早期去噪维持全局覆盖,提供多样化候选给后续奖励引导

**批大小缩放实验(Figure 4 right)**:
| 批大小K | 成功率 | Episode长度 | 推理时间(ms) |
|---------|--------|------------|-------------|
| 1 | 88% | 73.0 | 665 |
| 2 | 88% | 58.0 | 799 |
| 5 | 90% | 57.8 | 953 |
| 10 | 90% | 55.9 | 1239 |

发现:增大批大小提升性能但增加延迟,暴露计算-性能权衡,可根据部署需求调节。

**综合结论**:
- 梯度引导是核心,去除导致灾难性失败
- FK重采样和RBF多样性作为辅助,持续改善效率和鲁棒性
- 三者结合实现全局探索(RBF+FK)与局部精炼(梯度)的互补

**4.4 真实世界部署(Figure 5)**

**分布内任务(In-distribution)**:
- Level 1(单物体放置):VLS达到69%平均成功率,相比π0.5 baseline的50%提升**19%**
- Level 2(双物体顺序操作):VLS同样维持约69%,baseline约50%

**分布外任务(OOD)**:

| OOD类型 | 描述 | Baseline | VLS | 提升 |
|---------|------|----------|-----|------|
| 外观变化 | 用未见过的黄色盘子替换红/绿盘子 | 20% | 55% | +35% |
| 位置变化 | 交换两个盘子的位置,指令不变 | 25% | 60% | +35% |
| 物体替换 | 用未见过的马克杯替换香蕉 | 0% | 40% | +40% |

**关键发现**:
- 在最困难的物体级OOD(马克杯替换)中,baseline完全失败(0%),VLS仍有40%成功率
- VLS在真实物理不确定性下(物体滑动、部分执行)保持稳定
- 推理开销可接受(~1.2秒/动作块),证明可高效部署

**4.5 分析与讨论**

**为什么VLS有效?**
1. **语义接地**:VLM将抽象语言-观测对转化为具体的3D几何约束,使空间推理可操作化
2. **稠密引导**:轨迹级梯度提供细粒度的约束满足信号,远优于稀疏的离散反馈
3. **多模态互补**:梯度引导(局部精炼) + FK重采样(全局选择) + RBF多样性(早期覆盖)三管齐下
4. **阶段分解**:复杂任务分解为阶段特定势场,配合自适应引导和滞后切换,实现鲁棒的长期任务协调

**Surprising发现**:
- VLA的VLM骨干在后训练后泛化能力实际**降低**,与直觉相反——因为机器人数据entangle了空间推理与训练上下文
- 纯梯度引导(w/o FK)在简单任务仍可工作,但FK重采样在复杂多模态景观中至关重要
- 批大小K=2就能达到88%成功率,证明无需大批次就能有效steering

**局限性**:
1. **计算延迟**:批采样、MCMC运行、FK重采样引入高推理开销(1.2秒/块 vs baseline ~0.7秒)
2. **VLM依赖**:奖励函数质量依赖VLM的跨模态推理能力,若VLM生成错误的阶段分解或约束,VLS会失败
3. **可微性要求**:奖励函数必须可微,限制了某些复杂约束(如接触力、摩擦)的表达
4. **未测试极限**:未在超长期任务(>10阶段)或高度动态环境(快速移动物体)中验证

**未来方向**:
- 进度感知的奖励信号生成,根据执行进度动态调整奖励函数复杂度
- 优化推理效率,如早停、自适应批大小、并行FK重采样
- 扩展到非可微约束,如混合优化或约束松弛
- 与世界模型结合,预测性地调整引导策略
