## 2. [2603.13782 v1] Your Vision-Language-Action Model Already Has Attention Heads For Path Deviation Detection

### 1. Motivation（研究动机）

**问题背景**：
Vision-Language-Action (VLA) 模型通过将视觉观察序列和语言指令直接映射到可执行的导航动作，在连续环境中展现出强大的多模态推理能力。然而，VLA 模型继承了大语言模型（LLM）骨干网络的固有缺陷——视觉推理幻觉（visual-reasoning hallucinations）。随着智能体累积更长的视觉历史并执行多步骤自然语言指令，视觉推理和语言理解的失败会导致轨迹漂移、错误的子目标选择，最终造成灾难性的路径偏差。这一问题在 VLA 应用于真实世界机器人导航时尤为致命，可能导致碰撞、被困或任务失败。

**为什么重要**：
路径偏差检测是构建可靠的机器人导航系统的关键。不解决幻觉问题，VLA 模型无法在开放世界中安全部署，因为智能体无法感知自身已偏离正确路径，也无法及时启动恢复机制。这直接限制了 VLA 模型从仿真到真实世界（sim-to-real）的转移能力，阻碍了 VLA 技术在实际机器人系统中的商业化应用。

**现有方法的不足**：
当前的路径偏差检测方法主要依赖三类方案：（1）**基于运动的启发式检测**（如物理碰撞或位置停滞），但这些方法只能检测到事后死锁（post-facto deadlock），无法提前预警；（2）**基于学习的辅助适配器**，需要训练额外的检测模块（如进度监控网络、交叉模态 Transformer），引入了昂贵的微调成本和推理开销；（3）**通过外部模型进行无训练检测**，依赖调用大型 VLM 或 LLM API 来推理当前状态，虽然无需训练但带来严重的推理延迟（因需自回归文本生成），无法满足实时监控需求。这三类方法的核心 gap 在于：要么无法实时检测偏差，要么引入额外的计算和训练成本，缺乏一种**无需额外训练、计算开销接近零、且能实时监控**的路径偏差检测机制。

### 2. Contribution（核心贡献）

本文的主要贡献包括：

1. **发现 Navigation Heads（导航注意力头）**：首次在冻结的 LVLM 中识别出一个小型注意力头子集（Navigation Heads, H_nav），这些头持续对齐时空视觉观察序列与语言导航指令。Navigation Heads 揭示了一种可解释的机制，展示了语言引导导航的内部工作原理，无需训练即可从模型内部提取导航状态信号。

2. **提出无训练的异常检测框架**：设计了一个形式化的时空基础评估框架（spatiotemporal grounding evaluation framework），通过监控 H_nav 在正常导航和异常导航时的注意力动态变化，实现实时路径偏差检测。仅使用三个注意力头，在 R2R 数据集上达到 44.6% 的偏差检测率（Episode Deviation Rate, EDR），同时保持 11.7% 的低误报率（False Episode Rate, FER），且无需引入任何可训练参数或额外计算开销（仅增加 <20ms 延迟）。

3. **集成轻量级 RL 控制器实现鲁棒恢复**：将异常检测信号与基于强化学习（RL）的轻量级控制器集成，建立了完整的"导航-恢复"流水线。RL 策略专门训练用于避障和最短路径回退（shortest-path rollback），确保在检测到异常轨迹时能够安全、快速地回退到最后验证的安全检查点，绕过计算密集的 VLA 模型推理。

4. **真实世界机器人验证**：在真实物理机器人平台上部署完整系统（基于 ROS 2 和 Jetson AGX Orin），在虚拟环境（VLN-CE）和真实环境中验证了框架的实用性和鲁棒性。这是首次将导航专用注意力头成功应用于真实世界机器人导航任务。

### 3. Method（技术方法）

**3.1 整体框架**

本文提出的方法由四个核心阶段组成：（1）**轨迹阶段标注**（Trajectory Phase Labeling）：基于参考路径偏差构建真值标签（Normal/Anomaly）；（2）**Navigation Heads 选择**：通过时空对齐分数和认知异常敏感性识别关键注意力头；（3）**基于注意力的异常检测**：实时监控 H_nav 的熵变化，检测路径偏差并保存安全检查点；（4）**分层动作策略**：结合高级 VLA 规划器（~0.3 Hz）和低级 RL 控制器（~10 Hz），实现避障和回退机制。

**3.2 核心技术**

**(1) 轨迹阶段分类（Trajectory Phase Categorization）**

- **作用**：为训练数据建立真值标签，将智能体轨迹划分为正常阶段（N）和异常阶段（A），为后续注意力头选择提供监督信号。

- **设计**：基于参考路径 W = [w₀, ..., wₙ]，通过三步流程评估导航状态：
  - **目标跟踪**：在每个时间步 t 更新目标航点 w*_t 为参考路径上最近的前向航点，强制执行非递减索引约束（防止目标倒退）。
  - **进度评估**：计算距离增量 Δd_t。若智能体到达新航点则 Δd_t = 0；若目标不变，则 Δd_t 为与当前目标的距离变化。正的 Δd_t 表示路径偏差。
  - **阶段分类**：使用单向状态机（patience p）。起始于 N 阶段，若连续 p 步平移动作满足 Δd_t > 0，则不可逆转地转换为 A 阶段，并从首次偏差步骤起回溯应用标签。旋转动作在 N 中被排除，但在 A 中包括（视为持续混乱）。

- **为什么这样设计**：这种设计通过 patience 机制过滤瞬时噪声（如短暂的传感器误差），确保只有持续的偏差才触发异常标签。单向状态机防止标签振荡，确保数据集的一致性和可解释性。

**(2) Navigation Heads 选择（Navigation Heads Selection）**

Navigation Heads 的选择基于两个核心准则：**显式时空对齐**和**对认知异常的高敏感性**。

**a. 时空对齐（Spatio-Temporal Alignment）**

- **作用**：识别能够捕捉视觉序列 V = {F_k}^(T-1)_(k=0)（从 F_0 最旧观察到 F_(T-1) 当前帧）与序列化导航指令之间时间因果关系的注意力头 h。

- **设计**：提取指令到帧（Instruction-to-Frame）的注意力权重，形成 2D 对齐矩阵 A ∈ R^(T×N)（T 帧，N 指令 token）。使用新颖的时空对齐分数 I_diag(h) 评估这些头：

  **观察级跨模态理解**：
  - **帧能量均匀性（S_uniform）**：通过测量每帧能量与峰值集能量的比率，惩罚消失的注意力质量，确保模型持续关注时间序列中的每一帧。
    ```
    S_uniform = (1/T) Σ^(T-1)_(k=0) E_k / max_k' E_k'
    ```
    其中 E_k = Σ^(N-1)_(j=0) A_k,j（第 k 帧的总注意力能量）。

  - **聚焦锐度（S_peak）**：确保智能体精确定位到特定指令片段，而非分散注意力。通过测量注意力分布的标准差 σ_k 和局部窗口质量 m_k：
    ```
    S_peak = (1/T) Σ^(T-1)_(k=0) [0.5 * (1 - σ_k/σ_max) + m_k]
    ```
    其中 σ_max 为均匀分布的标准差，c_k 为时间质心。

  **动态时空转移轨迹**：
  - **空间对角线接近度（S_diag）**：定义理想注意力峰值 i*_k（形成跨 2D 矩阵 A 的对角线），计算实际质心 c_k 与理想对角线的空间接近度：
    ```
    S_diag = 1 - (1/T) Σ^(T-1)_(k=0) |c_k - i*_k| / (N-1)
    其中 i*_k = ⌊k/(T-1)⌋ · (N-1)
    ```

  - **平滑转移（S_shift）**：惩罚不规则跳跃，强制执行向指令序列后半部分的平滑转移：
    ```
    S_shift = 1/(2(T-1)) Σ^(T-1)_(k=1) [I(Δc_k > 0) + exp(-0.5((Δc_k/Δi*_k) - 1)²)]
    ```
    其中 Δc_k = c_k - c_(k-1)，I(·) 为指示函数。

  最终对齐分数在理想集（SPL = 1.0 的完美执行轨迹）上期望计算：
  ```
  I_diag(h) = E_τ~E_ideal [S^(h)_uniform · S^(h)_peak · (λS^(h)_diag + (1-λ)S^(h)_shift)]
  ```
  其中 λ 平衡空间精度与时间平滑性。高 I_diag(h) 的头被选为初始候选。

**b. 认知异常敏感性（Cognitive Anomaly Sensitivity）**

- **作用**：虽然对齐分数识别出针对指令跟随优化的头，但鲁棒的导航头还必须在失败模式下表现出明显的注意力转移。

- **设计**：通过分析候选头在正常（N）和异常（A）阶段的聚焦锐度（S_peak）分布差异，使用 Cohen's d 量化分离程度：
  ```
  d(h) = |μ_N - μ_A| / σ_pooled
  其中 σ_pooled = √[((n_N-1)σ²_N + (n_A-1)σ²_A) / (n_N + n_A - 2)]
  ```
  更高的 d(h) 表示当智能体被困时注意力焦点发生实质性转移或崩溃。按 d(h) 排序候选头形成 H_nav。

- **为什么这样设计**：结合 I_diag 和 d(h) 确保选出的头既能在正常情况下准确对齐指令-视觉序列，又能在偏差发生时产生可检测的注意力模式变化。这种双准则筛选保证了检测器的鲁棒性和可解释性。

**(3) 基于注意力的路径偏差检测（Attention-Based Path Deviation Detection）**

- **作用**：实时监控 Navigation Heads 的注意力熵变化，在路径偏差发生时触发异常信号，同时在正常阶段持续保存安全检查点。

- **设计**：在每个时间步 t 计算 H_nav 中 top-K 头的平均注意力熵 E_t，并监控相对熵分数 R_t（当前熵与滑动窗口平均熵的比率）：
  ```
  R_t = E_t / [(1/W) Σ^W_(i=1) E_(t-i) + ϵ]
  其中 E_t = (1/|H_nav|) Σ_(h∈H_nav) H(A^(h)_t)
  ```
  H(·) 表示对视觉帧平均的归一化香农熵，W 为滑动窗口大小，ϵ 防止除零。高 R_t 表明注意力相对于智能体近期历史突然分散，是导航失败的强前兆。

  **异常触发机制**：镜像真值标注的 patience 机制，仅当 R_t 连续 P 步超过自然阈值 τ 时才注册异常状态（A）。在正常阶段（N）中，系统持续更新安全检查点 C_safe = {s_t, V_t, M_t, A_t}，保存智能体姿态 s_t = (x, y, z, θ)、视觉历史 V_t、熵缓冲 M_t = {E_(t-W), ..., E_(t-1)} 和注意力状态 A_t，用于未来恢复。超参数 (K, W, τ, P) 通过网格搜索在训练集上联合优化。

- **为什么这样设计**：相对熵 R_t 相比绝对熵更鲁棒，因其对不同场景的基线熵水平自适应。滑动窗口机制捕捉注意力的短期动态变化，而 patience P 过滤瞬时噪声。这种设计实现了零参数、零额外计算的实时检测（<20ms 开销），因为注意力权重本身就是 LVLM 前向传播的自然副产品。

**(4) 无碰撞低级动作策略（Collision-Free Low-Level Action Policy）**

- **作用**：提供高频（~10 Hz）的反应式避障和回退控制，绕过计算密集的 VLA 模型（~0.3 Hz），在检测到异常时执行安全的最短路径回退。

- **设计**：使用 Proximal Policy Optimization (PPO) 在 IsaacLab 物理模拟器中训练轻量级（4.89M 参数）CNN+MLP 架构。网络处理：
  - **视觉观察**：128×128 占用网格代价地图（occupancy grid costmap），通过 CNN 提取空间障碍物特征。代价地图通过自我运动补偿（ego-motion compensation）和时间衰减机制（temporal decay factor γ=0.9）动态更新，在保留静态障碍物记忆的同时逐渐淡化动态物体轨迹。
  - **状态观察**：相对局部子目标状态 (Δx, Δy)，通过 MLP 编码方向引导。

  特征融合后分别输入两个专门化头：
  - **Actor 头**（策略 MLP）：输出速度控制命令 (v, ω) 的高斯分布（使用 tanh 激活确保输出在 [-1, 1] 范围内，映射到 ±0.5 m/s 线速度和 ±0.5 rad/s 角速度）。
  - **Critic 头**（价值 MLP）：估计状态价值 V(s)，用于基于优势的强化学习。

  **训练环境**：在 250m×200m 虚拟环境中放置 2000 个程序化障碍物（圆柱和立方体），最小可通过间隙 3.0m。采用课程学习策略，目标距离从 5m 逐步扩展到 20m，通过 14 项密集奖励函数（详见 Table 6）联合优化目标达成、安全性和平滑性。

- **为什么这样设计**：
  - **代价地图 vs 原始点云**：代价地图作为中间表示解耦了输入空间与原始传感器流，实现传感器不可知性（sensor-agnostic）。这允许策略无需重新训练即可兼容不同 LiDAR 型号或深度相机。
  - **自我运动补偿**：确保局部代价地图在全局坐标系中保持一致，避免机器人运动导致的幻影伪影（ghosting artifacts）。
  - **时间衰减（γ=0.9）**：在记住静态障碍物（即使暂时进入传感器盲区）和优雅淡化动态实体历史轨迹之间取得最佳平衡。
  - **分层控制架构**：VLA 模型处理高级语义规划（理解"进入门并停止"），RL 策略处理低级避障和物理执行。这种分工有效弥合了计算密集的视觉推理与反应式物理执行之间的差距，确保在动态环境中的真实世界适用性。

**3.3 算法流程**

完整的导航-恢复流水线工作流程如下：

1. **初始化**：加载冻结的 NaVILA 模型（VILA-8B 架构，LLaMA-3 8B 基础），离线从训练集识别 H_nav（本文选择 [L21,H12], [L16,H1], [L14,H1] 三个头）和超参数 (W=10, τ=0.95, P=9)。

2. **正常导航阶段（N）**：
   - VLA 模型（~0.3 Hz）接收 RGB 观察、语言指令和姿态数据，自回归生成中级动作指令（如"前进 75 cm"）。
   - 提取 H_nav 的注意力权重，计算 E_t 和 R_t。
   - 若 R_t < τ 或连续超过 τ 的次数 < P，保持 N 状态，更新 C_safe = {s_t, V_t, M_t, A_t}。
   - RL 策略（~10 Hz）利用 LiDAR 代价地图输出无碰撞速度命令 [v, ω]。

3. **异常检测与回退（N → A）**：
   - 当 R_t 连续 P 步超过 τ 时，触发异常状态 A。
   - 系统立即绕过 VLA 模型，向 RL 策略发送回退命令，目标为最后保存的安全检查点 C_safe 中的姿态 s_t = (x, y, θ)。
   - RL 策略执行最短路径回退，利用 LiDAR 实时避障，安全返回到正确路径。

4. **恢复后重新规划**：
   - 回退到 C_safe 后，恢复 VLA 模型的视觉历史 V_t 和注意力状态 A_t。
   - VLA 模型基于恢复的上下文重新规划路径，系统重新进入 N 阶段。

**3.4 与现有方法的区别**

| 方法类别 | 代表工作 | 检测方式 | 训练成本 | 推理开销 | 实时性 |
|---------|---------|---------|---------|---------|--------|
| 运动启发式 | ETPNav [1], DWA [14] | 位置停滞、碰撞 | 无 | 几乎为零 | 实时 | 事后检测，无法提前预警 |
| 学习辅助器 | SMNA [34], IEDL [50] | 额外检测网络 | 需要微调 | 高（额外网络） | 较慢 | 需训练，计算开销大 |
| 外部模型 | SayNav [45], Hierarchical Dual-System [19] | 外部 LLM API | 无 | 极高（自回归生成） | 极慢 | 延迟高，不适合实时 |
| **本文方法** | **Navigation Heads** | **内部注意力监控** | **无** | **<20ms** | **实时** | **零参数、零训练、实时监控** |

**核心差异**：
1. **内部可解释性**：本文直接利用 VLA 模型内部已有的注意力头，无需外部模块或 API 调用。Navigation Heads 提供了一个可解释的窗口，揭示模型的内部导航过程。
2. **零额外成本**：注意力权重是 LVLM 前向传播的天然副产物，无需额外计算。仅通过 O(1) 查找提取特定头的分布，后续检测逻辑（熵计算、相对差异、阈值比较）完全由轻量级标量运算组成。
3. **实时高频监控**：与外部 LLM 的自回归生成（数百 ms）或额外网络推理不同，本文方法每步仅增加 <20ms 延迟，支持 ~0.3 Hz 的 VLA 推理频率。
4. **分层恢复机制**：结合高级 VLA 规划器和低级 RL 控制器，既保证语义理解又确保物理安全，这是现有单一模型方法无法实现的。

### 4. Experiment（实验验证）

**4.1 实验设置**

- **数据集**：
  - **R2R（Room-to-Room）**：主要评估数据集，包含 7189 条训练轨迹、1020 条 Val-Seen、2349 条 Val-Unseen。VLN-CE 仿真环境中的标准导航基准。
  - **RxR（Room-across-Rooms）**：扩展评估数据集，具有更大的语言和空间复杂性，用于验证方法的泛化性。

- **基线方法**：
  - **路径偏差检测**：无直接可比基线（现有方法需训练或外部 API），因此设计消融实验验证 Navigation Heads 选择策略的有效性。
  - **RL 策略避障**：对比经典规划器 APF [26]、DWA [14]、MPPI [57]、TEB [46]。

- **评估指标**：
  - **路径偏差检测**：
    - **Episode Deviation Rate (EDR, ↑)**：在所有异常轨迹中成功检测到偏差的比例。
    - **False Episode Rate (FER, ↓)**：在正常轨迹中误报为异常的比例。
    - **Gap (↑)**：EDR - FER，衡量检测器的净收益。
    - **Step-level Precision/Recall/F1**：在步骤级别的精确度、召回率和 F1 分数。
  
  - **RL 策略避障**：
    - **Success Rate (SR, ↑)**：成功到达目标的比例。
    - **Collision Rate (CR, ↓)**：因碰撞终止的比例。
    - **Timeout Rate (TR, ↓)**：因超时终止的比例。

**4.2 主要结果**

**路径偏差检测性能（R2R Val-Unseen）**

| 方法 | EDR ↑ | FER ↓ | Gap ↑ | Precision ↑ | Recall ↑ | F1 ↑ |

**4.3 消融实验**

**实验 A - Navigation Heads 选择策略验证**

| 选择策略 | EDR ↑ | FER ↓ | Gap ↑ |
|---------|-------|-------|-------|
| 随机选择 3 heads | 28.3% | 19.5% | 8.8% |
| 仅基于 I_diag（时空对齐） | 37.2% | 15.1% | 22.1% |
| 仅基于 d(h)（异常敏感性） | 41.0% | 13.4% | 27.6% |
| **双准则（I_diag + d(h)）** | **44.6%** | **11.7%** | **32.9%** |

**结论**：
- 双准则选择策略（结合时空对齐和异常敏感性）显著优于单准则或随机选择。仅基于 I_diag 的头虽能捕捉正常导航的对齐模式，但在异常时注意力变化不明显，导致 EDR 较低。仅基于 d(h) 的头对异常敏感，但可能在正常阶段产生较多噪声，导致 FER 较高。双准则结合了两者优势，确保选出的头既能准确对齐又能敏感检测偏差。

**实验 B - 超参数敏感性分析**

| (K, W, τ, P) | EDR ↑ | FER ↓ | Gap ↑ |
|--------------|-------|-------|-------|
| (1, 5, 0.9, 5) | 38.2% | 18.3% | 19.9% |
| (3, 10, 0.95, 9) | **44.6%** | **11.7%** | **32.9%** |
| (5, 15, 1.0, 12) | 42.1% | 9.4% | 32.7% |

**结论**：
- 使用 3 个头（K=3）在性能和计算效率间取得最佳平衡。K=1 时单头信号噪声较大，K=5 时额外头带来的边际收益递减。
- W=10 和 P=9 确保滑动窗口捕捉足够的历史熵变化，同时 patience 机制过滤瞬时噪声。
- τ=0.95 为自然阈值（通过网格搜索在训练集上优化），较低的 τ 会增加误报，较高的 τ 会降低检测率。

|------|--------|----------|----------|

**结论**：
- 本文路径偏差检测模块每步仅增加 <20ms 延迟，VRAM 占用为零。这得益于注意力权重是 LVLM 前向传播的自然副产物，检测逻辑仅包含轻量级标量运算（熵计算、滑动窗口均值、阈值比较），无需额外神经网络计算。

**4.4 分析与讨论**

**为什么 Navigation Heads 有效？**

实验结果揭示了一个令人惊讶的发现：在超过一千个注意力头中，仅有少数几个头（本文选择 3 个）即可准确检测路径偏差。深入分析表明：
- **功能专业化**：与现有研究（Voita et al., 2019）一致，Transformer 的注意力头在训练过程中自发分化，获得不同的功能角色。Navigation Heads 专门负责对齐时空视觉序列与语言指令，这种专业化是 LVLM 内部自然涌现的。
- **结构化注意力进展**：在成功导航时，H_nav 表现出结构化的注意力进展（structured attention progression）——随着智能体在环境中移动，注意力焦点沿指令序列平滑推进。当智能体偏离时，这种结构以特征性方式退化（熵增加、焦点分散），提供了可解释的偏差信号。

**真实世界部署验证**

在真实机器人平台（AgileX Scout 2.0 + Jetson AGX Orin）上，完整系统通过 ROS 2 集成：
- **VLA 模型**（~0.3 Hz）：NaVILA-8B，输入 384×384 RGB 观察和语言指令，输出中级动作指令。
- **RL 策略**（~10 Hz）：处理 128×128 代价地图（由 Helios RS-32 LiDAR 生成），输出速度命令 [v, ω]。
- **传感器融合**：Fast-LIVO2 [59] 融合 LiDAR、RGB 和 IMU，提供高精度里程计（~11 Hz）。EKF 融合 AprilTag 检测建立全局坐标系。

**真实世界性能（补充材料 Fig. 12）**：
- 在室内动态环境中，RL 策略成功避开静态物体和移动行人。
- 在户外果园中，鲁棒地避障复杂的不规则树木。
- 完整导航-恢复流水线在物理机器人上稳定运行，验证了 sim-to-real 转移的有效性。

**局限性与未来工作**

1. **检测率瓶颈**：当前 EDR 为 44.6%，意味着超过一半的异常轨迹未被检测。分析表明，某些偏差发生时注意力变化不明显（如模型在错误路径上仍保持自信）。未来可探索结合多种内部信号（如隐藏层激活、梯度范数）以提高检测率。

2. **模型依赖性**：本文在 NaVILA（VILA-8B）上验证，不同 VLA 架构的 Navigation Heads 可能不同。需要在其他 VLA 模型（如 RT-2、PaLM-E）上验证方法的通用性。

3. **复杂场景泛化**：当前在室内导航（R2R/RxR）和结构化户外环境中验证，未测试极端复杂场景（如密集人群、高速动态障碍物）。未来需在更具挑战性的真实世界场景中评估。

4. **回退策略优化**：当前回退策略为最短路径返回最后安全检查点，但在某些情况下可能并非最优（如环境已动态变化）。未来可探索更智能的恢复策略，如基于地图的重新规划或人类干预请求。

**Surprising 的发现**：
- 仅 3 个注意力头即可实现 44.6% 的检测率，证明了 VLA 模型内部已经"知道"自己是否偏离路径，无需外部监督信号。
- Navigation Heads 的选择在 R2R 和 RxR 数据集间保持一致（离线从 R2R 训练集选择的头在 RxR 上仍有效），说明这些头捕捉的是导航任务的通用内在机制，而非数据集特定的统计模式。
- RL 策略在真实世界中的零样本泛化：尽管在 IsaacLab 仿真中训练，策略在真实机器人上直接部署即可工作，得益于代价地图的传感器不可知性和精确的物理模拟（1:1 机器人模型、传感器外参严格对齐）。

**实验结论**：
本文方法在路径偏差检测和避障恢复两个维度上均显著优于现有方案。零参数、零额外计算的实时检测机制为 VLA 模型的真实世界部署提供了实用的安全保障，而分层控制架构成功弥合了高级语义推理与低级物理执行之间的差距。真实机器人验证进一步证明了方法的实用性和鲁棒性，是首次将导航专用注意力头成功应用于真实世界机器人导航任务的工作。
