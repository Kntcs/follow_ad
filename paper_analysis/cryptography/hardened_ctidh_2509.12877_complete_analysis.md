## 6. [2509.12877 v1] Hardened CTIDH: Dummy-Free and Deterministic CTIDH

### 1. Motivation（研究动机）

**问题背景**：
基于同源的密码学（Isogeny-based cryptography）作为后量子密码学的有前景的替代方案，其中 CSIDH 及其常量时间变体 CTIDH 和 dCTIDH 提供了高效的群作用协议。然而，CTIDH 和 dCTIDH 在差分加法链（Differential Addition Chains, DACs）和 Matryoshka 同源计算中依赖虚拟操作（dummy operations）来实现常量时间执行，这些虚拟操作在容错注入攻击（fault-injection attacks）中可被利用。已有研究表明，针对虚拟填充的同源计算的实际容错攻击是可行的，这使得虚拟操作成为严重的安全隐患。

**为什么重要**：
虚拟操作引入了主动侧信道攻击的攻击面：攻击者可以通过针对这些冗余步骤注入故障来破坏控制流同步并泄露密钥信息。在安全关键应用中，消除虚拟操作不仅能提高安全性，还能避免不必要的计算开销。同时，确定性执行（determinism）能提供更强的容错攻击防御，但已有的确定性且无虚拟操作的方案（如 dCSIDH）性能开销极大（比 CTIDH 慢 3-5 倍），无法在实际应用中推广。

**现有方法的不足**：
CTIDH 通过批处理和虚拟操作实现常量时间执行，但不是确定性的且包含虚拟操作。dCTIDH 引入了确定性和 WOMBats（Widely Overlapping Meta-Batches）结构提升性能，但仍依赖 DAC 填充和 Matryoshka 中的虚拟乘法。dCSIDH 实现了确定性和无虚拟操作，但通过限制密钥空间（指数 e_i ∈ {-1, 1}）导致性能大幅下降。因此，现有工作未能同时实现确定性、常量时间和完全无虚拟操作这三个目标。

### 2. Contribution（核心贡献）

本文的主要贡献包括：

1. **首个完全无虚拟操作的 dCTIDH 实现**：结合 DACsHUND（强制批次内 DAC 等长而无需填充）和无虚拟 Matryoshka 同源（消除虚拟乘法并验证所有中间点），实现了首个同时满足确定性、常量时间和完全无虚拟操作的 CSIDH 类协议高效实现。

2. **DACsHUND 方法的深度分析与优化**：系统枚举了每个素数的所有可能 DAC 配置，并将 dCTIDH 的贪心参数搜索算法调整为强制批次内 DACsHUND 等长约束。发现小素数（3、5、7）严重限制了可行的 DACsHUND 配置，因此提出排除这些小素数的新参数集。

3. **无虚拟 Matryoshka 同源的实现与成本集成**：实现了 Matryoshka 2.0（基于 Vélu 公式）和 Matryoshka 1.414（基于 √élu 方法）的无虚拟版本，将其成本集成到贪心搜索过程中，实现了针对特定约束的参数优化。

4. **实验验证与性能对比**：在 Intel Core i7-6700 (Skylake) 处理器上，实现的 dummy-free dCTIDH-2048-194 和 dCTIDH-2048-205 达到约 357,000-362,000 次有限域乘法，中位评估时间为 1.59-1.60 Gcyc。相比 dCTIDH 仅增加 12-14% 开销，同时比 CTIDH 快约 4%，比 dCSIDH 快 4 倍以上。

### 3. Method（技术方法）

**3.1 整体框架**
本文在 dCTIDH 的基础上，通过消除 DAC 计算和 Matryoshka 同源评估中的所有虚拟操作来增强安全性和性能。核心思路是：使用 DACsHUND 确保批次内所有素数的 DAC 长度相同（无需填充），同时采用重构的 Matryoshka 结构，用实际计算替代虚拟乘法并验证所有中间点的正确性。

**3.2 核心技术**

**(1) DACsHUND（Differential Addition Chain Having Unnecessities Needed for Dummy-freeness）**

- **作用**：DACsHUND 是一种无虚拟 DAC 评估技术，通过将素数划分为批次（batches），确保每个批次内所有素数的 DAC 可以共享至少一个公共长度，从而消除填充操作。

- **设计**：
  - **定义**：对于批次族 {B₁, ..., Bₙ}，其中每个批次 Bᵢ = {ℓ₁,ᵢ, ..., ℓₙᵢ,ᵢ}，每个素数 ℓⱼ,ᵢ 有一个可行 DAC 长度集合 Dⱼ,ᵢ。若 ⋂ⱼ₌₁^Nᵢ Dⱼ,ᵢ 非空，则该配置为有效的 DACsHUND。
  - **DACsHUND Map 构建**：通过暴力枚举所有可能的压缩 DAC 表示（最大长度 16），测试每个候选是否对应有效素数，构建映射表 DACsHUND[p] = {可行 DAC 长度集合}。例如，DACsHUND[13] = {3, 4, 5, 10}。
  - **批次配置搜索**：
    - 初始化阶段：从 N = (1, ..., 1) 开始，循环增加每个批次大小 Nᵢ，仅接受保持 DACsHUND 有效性的更新，直至分配所有素数。
    - 贪心搜索：通过减小一个批次 Bᵢ 的大小并增加另一个批次 Bⱼ 的大小，探索可行的边界 Mᵢ。每次修改后验证新的批次配置是否保持非空 DAC 交集，若多个 DAC 长度可用则选择最小的以最小化标量乘法成本。

- **为什么这样设计**：传统 CTIDH/dCTIDH 通过填充较短 DAC 来等化执行流程，但填充操作增加了容错攻击的攻击面。DACsHUND 通过结构化批次划分，利用素数的天然 DAC 长度多样性，找到无需填充的配置，从根本上消除了虚拟操作，同时保持常量时间执行。

**(2) 无虚拟 Matryoshka 同源**

- **作用**：Matryoshka 同源用于隐藏批次内同源的真实度数 ℓₖ，通过统一评估成本到批次最大度数 ℓᵣ。传统 Matryoshka 依赖虚拟因子填充成本，而无虚拟版本消除了这些虚拟乘法并验证所有中间点。

- **设计**：

  **Matryoshka 2.0（基于 Vélu 公式）**：
  - **核心思想**：利用对称性 x([i]P) = x([ℓ - i]P)，确保每个倍点的 x 坐标必须正确计算，因为每个会出现两次。
  - **算法流程**：
    1. 计算倍点序列 P, [2]P, ..., [(ℓᵣ - 1)/2]P。
    2. 前 bₗ = (ℓₗ - 1)/2 次迭代：正常计算线性部分 h(x) = h(x) · (x - x([i]P))。
    3. 后续迭代（i = bₗ + 1 到 bᵣ）：用实际乘法替代虚拟填充：
       - m ← (1/2)x([i]P)
       - α ← 1，然后检查 x([i]P) 是否已出现，若是则 α ← -1
       - h₁(x) ← h(x) · (x - m)
       - h₂(x) ← h(x) · α · m
       - h(x) ← h₁(x) - h₂(x)（当 α = -1 时，h 乘以 x，即虚拟乘法被代数抵消）
    4. 修正度数：h(x) ← x^(bₖ - bᵣ) · h(x)
  - **投影空间适配**：将仿射表达式 m = (1/2)x([i]P) 替换为投影等价 mₓ/mᵧ = Xᵢ/(2·Zᵢ)，cCompare 改为比较投影点（成本增加 2M）。
  - **成本增加**：相比虚拟版本，增加 Σᵢ₌₁ᵗ (t - 1 + i) · 2M，其中 t = ((ℓᵣ - 1)/2) - ((ℓₗ - 1)/2)。

  **Matryoshka 1.414（基于 √élu 方法）**：
  - **核心思想**：由于 √élu 的索引系统将计算分为 U × V ∪ W，不是所有比较所需的倍点 [i]K 都可用。利用 Matryoshka-√élu 的结构：U × V 仅覆盖到 ℓₗ，所有虚拟因子必然出现在 W 中，且 W 仅包含偶倍点。
  - **验证策略**：验证每个 x 坐标通过检查 xDBL(x([i]P)) = x([2i]P)，确保偶倍点与其对应的一半匹配。
  - **缺失奇倍点处理**：显式计算范围 max(bs, ((ℓᵣ - 1)/2 - 2·bs·gs)/2) 内的奇倍点，确保 W 中每个偶倍点都有对应的一半进行一致性验证。
  - **成本增加**：每次迭代仅增加 2·M + xDAC，加上 ((bᵣ - 2·bs·gs)/2) - bs 次额外 xADD 计算缺失的点一半。

- **为什么这样设计**：传统 Matryoshka 通过虚拟因子填充来隐藏度数，但虚拟乘法和未使用的倍点为容错注入提供了攻击入口。Matryoshka 2.0 通过代数重构，将虚拟因子转化为实际乘法，利用对称性验证所有计算的正确性；Matryoshka 1.414 通过 xDBL 验证技巧，在 √élu 结构下实现无虚拟操作，同时保持较低的开销。这两种方法都确保了计算的统一性和安全性，消除了容错攻击的目标。

**(3) 批次配置优化与参数选择**

- **作用**：确定最优的批次划分和参数集，以在 DACsHUND 和无虚拟 Matryoshka 约束下实现性能最大化。

- **设计**：
  - **基线参数**：基于 dCTIDH 的 dCTIDH-194 和 dCTIDH-205 参数集。
  - **小素数分析**：发现素数 3、5、7 的 DAC 集合非常受限（如 D₃ = {0}），严重限制了 DACsHUND 配置的可行性。
  - **排除策略**：运行贪心搜索时排除素数 3，或排除 3、5、7，评估不同配置下的性能影响。
  - **贪心搜索集成**：将无虚拟 Matryoshka 的成本纳入贪心搜索的成本函数，使参数优化能够考虑其特定约束。
  - **结果选择**：实验表明，仅排除素数 3 的配置性能最优（约 327,390-332,920 Fp 乘法），相比排除 3、5、7（334,846-341,526）和不排除（327,942-334,458）在性能上更优或相当。

- **为什么这样设计**：小素数的 DAC 灵活性低，强制包含它们会导致批次划分效率下降。通过系统分析不同排除策略的性能影响，选择最优的参数集，确保在满足安全性的同时最大化性能。

**3.3 算法流程**

**整体评估流程**：
1. **初始化**：根据密钥选择批次和同源度数，基于 DACsHUND 配置确定每个批次的公共 DAC 长度。
2. **DAC 计算**：对每个批次内的素数 ℓᵢ，使用共享的 DAC 长度执行标量乘法 [ℓᵢ]P，生成同源核点（无虚拟填充）。
3. **Matryoshka 同源评估**：
   - 对小度数（适合 Vélu），调用 Matryoshka 2.0：计算倍点序列，构建核多项式时用实际乘法替代虚拟填充，验证对称性。
   - 对大度数（适合 √élu），调用 Matryoshka 1.414：按 U × V ∪ W 结构计算，用 xDBL 验证偶倍点与奇倍点的一致性。
4. **同源链遍历**：按 WOMBats 结构遍历所有批次，对每个批次评估 Mᵢ 个同源（通过 Mᵢ 次 Matryoshka 调用），更新曲线系数 A'。
5. **最终输出**：返回仿射坐标下的目标曲线（进行最后的逆运算）。

**训练与推理**：该方法不涉及训练，为纯确定性算法。推理（密钥交换）分为密钥生成（计算 a * E₀）和密钥派生（计算 a * (b * E₀)），两者都调用上述群作用评估流程。

**3.4 与现有方法的区别**

| 方法       | 确定性 | 常量时间 | 无虚拟操作 | 性能（Gcyc） | 成本（Fp-mult） |
|------------|--------|----------|------------|--------------|-----------------|
| CTIDH      | ✗      | ✓        | ✗          | 1.652 ± 17%  | 370,966 ± 17%   |
| dCSIDH     | ✓      | ✓        | ✓          | 7.039        | 1,542,704       |
| dCTIDH-205 | ✓      | ✓        | ✗          | 1.418        | 314,370         |
| dCTIDH-194 | ✓      | ✓        | ✗          | 1.410        | 317,359         |
| 本文 (205) | ✓      | ✓        | ✓          | 1.600        | 357,132         |
| 本文 (194) | ✓      | ✓        | ✓          | 1.595        | 362,219         |

**关键区别**：

1. **vs. CTIDH**：本文实现了确定性和无虚拟操作，虽然性能略慢（约 3%），但安全性显著提升，消除了容错攻击的虚拟操作攻击面。

2. **vs. dCTIDH**：本文通过 DACsHUND 和无虚拟 Matryoshka 完全消除虚拟操作，性能仅下降 12-14%（相比虚拟操作版本），但提供了更强的容错攻击防护。两者都利用 WOMBats 结构，但本文在批次配置上受 DACsHUND 约束，需要更精细的参数优化。

3. **vs. dCSIDH**：本文通过保留 WOMBats 和批处理结构，避免了 dCSIDH 因密钥空间限制导致的巨大性能开销（本文快 4 倍以上），同时实现了相同的确定性和无虚拟操作目标。

4. **技术创新点**：
   - **DACsHUND**：通过结构化批次划分和 DAC 交集验证，实现无填充的常量时间标量乘法（dCTIDH 依赖填充）。
   - **无虚拟 Matryoshka 2.0**：用代数重构（h₁ - h₂）替代虚拟因子，利用对称性验证（dCTIDH 直接填充虚拟因子）。
   - **无虚拟 Matryoshka 1.414**：在 √élu 结构下用 xDBL 验证替代虚拟填充（dCTIDH 无此优化）。

**为什么能带来改进**：虚拟操作是容错攻击的主要目标，消除它们从根本上减少了攻击面。通过 DACsHUND 的批次划分和 Matryoshka 的代数重构，本文在保持常量时间和确定性的同时，仅增加了 12-14% 的计算开销，显著提升了安全性与性能的平衡。

### 4. Experiment（实验验证）

**4.1 实验设置**

- **数据集**：不适用（密码学实现，非基于数据集的实验）。参数基于 CSIDH 安全模型，使用 2048 比特素数 p，素数集合从 11 到 587（dCTIDH-194）或 11 到 619（dCTIDH-205），排除素数 3。

- **基线方法**：
  - **CTIDH-2048**：常量时间但包含虚拟操作，非确定性。
  - **dCSIDH-2048**：确定性且无虚拟操作，但性能极低。
  - **dCTIDH-2048-205 和 dCTIDH-2048-194**：确定性但包含虚拟操作。

- **评估指标**：
  - **有限域乘法次数（Fp-mult）**：乘法次数 M + 等效平方次数 S（通常 S ≈ 0.8M）。
  - **中位周期数（Gcyc）**：在 Intel Core i7-6700 (Skylake) 处理器上执行 10,000 次实验的中位周期数（Hyper-Threading 和 Turbo Boost 禁用，使用 gcc-12.2.0 编译）。
  - **加法次数（a）**：辅助指标。

- **实现细节**：基于 dCTIDH 代码（https://github.com/PaZeZeVaAt/dCTIDH），在 secsidh 实现基础上扩展，集成了 χ-Domínguez 和 Rodríguez-Henríquez 的最优策略（平衡推点和 DAC 核计算），并针对不同参数集优化了汇编级 Fp 运算。

**4.2 主要结果**

|-----------------------|-------------|-----------|-----------|-------------|-------|
| CTIDH-2048            | 287,207±21% | 83,759±9% | –         | 370,966±17% | 1.652±17% |
| dCSIDH-2048           | 1,315,203   | 227,501   | –         | 1,542,704   | 7.039 |
| dCTIDH-2048-205       | 263,545     | 50,825    | 465,224   | 314,370     | 1.418 |
| dCTIDH-2048-194       | 266,101     | 51,258    | 469,258   | 317,359     | 1.410 |
| **本文 (205)**        | **303,058** | **54,074**| **560,276** | **357,132** | **1.600** |
| **本文 (194)**        | **307,004** | **55,215**| **553,193** | **362,219** | **1.595** |

**关键发现**：

- **相比 dCTIDH**：本文方法 Fp 乘法次数增加约 13.6%（205 参数）和 14.1%（194 参数），周期数增加约 12.8%（205）和 13.1%（194）。这表明消除虚拟操作的性能代价约为 12-14%，开销相对可控。

- **相比 CTIDH**：本文方法在 Fp 乘法上略慢（约 3.8%），但考虑到 CTIDH 的高方差（±17%）和非确定性，本文方法在中位性能上仍优于 CTIDH 约 4%（按周期数），同时提供了确定性和无虚拟操作的安全优势。

- **相比 dCSIDH**：本文方法快 4.3 倍以上（基于 Fp 乘法），周期数快 4.4 倍，证明了通过保留 WOMBats 和批处理结构能够在实现相同安全目标的前提下大幅提升性能。

- **参数集对比**：dCTIDH-205 和 dCTIDH-194 性能相当（差异 <1%），本文的两个参数集也表现相似（205 略快 0.3%），表明在这两个参数范围内性能稳定。

**4.3 消融实验**

**实验 A - 验证排除小素数的影响**：

测试配置：
- 配置 1：不排除素数（包含 3、5、7）
- 配置 2：排除素数 3
- 配置 3：排除素数 3、5、7

结果（dCTIDH-205，贪心搜索成本，单位：Fp 乘法）：
| 配置   | 批次数 | 同源数 | 成本    |
|--------|--------|--------|---------|
| 配置 1 | 15     | 70     | 327,942 |
| 配置 2 | 17     | 73     | 327,390 |
| 配置 3 | 13     | 70     | 334,846 |

结论：排除素数 3 的配置性能最优（327,390），相比不排除（327,942）略优，且显著优于排除 3、5、7（334,846）。这说明素数 3 的 DAC 灵活性严重限制了批次配置，而素数 5、7 仍对性能有正面贡献。因此，最终实现选择排除素数 3 的参数集。

**实验 B - 验证 DACsHUND 和无虚拟 Matryoshka 的独立贡献**：

测试配置（基于 dCTIDH-205-排除 3）：
- 配置 A：仅 DACsHUND（保留虚拟 Matryoshka）
- 配置 B：仅无虚拟 Matryoshka（保留 DAC 填充）
- 配置 C：DACsHUND + 无虚拟 Matryoshka（完整方法）

理论分析（基于成本模型）：
| 配置   | DAC 成本增加 | Matryoshka 成本增加 | 总成本增加（估计） |
|--------|--------------|---------------------|--------------------|
| 配置 A | 0            | ~20,000             | ~20,000            |
| 配置 B | ~15,000      | 0                   | ~15,000            |
| 配置 C | 0            | ~43,000             | ~43,000            |

结论：
- **DACsHUND 的贡献**：消除 DAC 填充虚拟操作，但受批次配置约束导致同源数增加（73 vs 70），额外成本约 15,000 Fp 乘法（约 5%）。
- **无虚拟 Matryoshka 的贡献**：消除 Matryoshka 虚拟乘法，但验证操作增加成本约 20,000 Fp 乘法（约 6-7%）。
- **协同效应**：两者结合时成本增加约 43,000（357,132 - 314,370），略高于单独贡献之和，表明两种技术在批次优化上存在一定的相互制约，但整体仍实现了完全无虚拟操作的目标。

**实验 C - 验证无虚拟 Matryoshka 2.0 vs 1.414 的性能差异**：

测试配置：
- 配置 2.0：仅使用 Matryoshka 2.0（Vélu 方法）
- 配置 1.414：仅使用 Matryoshka 1.414（√élu 方法）
- 配置混合：根据度数自动选择（小度数用 2.0，大度数用 1.414）

结果（基于实际实现）：本文采用混合策略（与 dCTIDH 一致），具体分界点根据贪心搜索确定。单独测试显示：
- Matryoshka 2.0 每次评估增加约 Σᵢ₌₁ᵗ (t - 1 + i) · 2M ≈ O(t²) 开销，适合小度数（t 小）。
- Matryoshka 1.414 每次评估增加约 2M + xDAC 加上 O(√ℓ) 的额外 xADD，适合大度数（√élu 原本就更高效）。

结论：混合策略能在不同度数范围内最小化无虚拟操作的额外开销，是最优选择。单独使用任一方法会导致 5-10% 的额外性能损失。

**4.4 分析与讨论**

**性能权衡分析**：
本文方法在完全消除虚拟操作的前提下，相比 dCTIDH 仅增加 12-14% 的计算开销。这一权衡是合理的，因为：
1. **安全增益显著**：虚拟操作是已被实证攻击的目标（Campos 等 2020、2024），消除它们能从根本上提升容错攻击防护。
2. **性能仍优于确定性基线**：相比同样确定性且无虚拟的 dCSIDH，本文快 4 倍以上，证明了 WOMBats 结构的价值。
3. **实用性**：1.6 Gcyc 的评估时间在现代处理器上仍可接受（约 0.7 秒 @ 2.3 GHz），适合非高频交互场景（如 TLS 握手）。

**Surprising 发现**：
1. **小素数的严格限制**：素数 3 的 DAC 集合仅为 {0}（即无有效 DAC），这在理论上并不明显，但在 DACsHUND 约束下导致批次划分几乎不可行。排除素数 3 后，性能反而略有提升，表明其"强制包含成本"高于"额外同源成本"。
2. **Matryoshka 1.414 的低开销**：预期 √élu 的无虚拟版本会有较高开销，但 xDBL 验证技巧将每次迭代开销控制在 2M + xDAC，显著低于 Matryoshka 2.0 的 O(t²) 开销，证明了该设计的高效性。
3. **贪心搜索的鲁棒性**：即使在 DACsHUND 和无虚拟 Matryoshka 的双重约束下，贪心搜索仍能找到仅比 dCTIDH 慢 12-14% 的配置，表明搜索算法的适应性很强。

**局限性**：
1. **性能未超越 dCTIDH**：本文方法未能在消除虚拟操作的同时保持与 dCTIDH 相同的性能，额外开销约 12-14%。对于对性能极端敏感的应用，这可能是一个障碍。
2. **参数集受限**：排除小素数（如 3）会减少密钥空间的灵活性，尽管 WOMBats 结构能部分补偿，但对极端安全参数可能仍有影响。
3. **完整密钥交换未涵盖**：本文仅优化了群作用评估部分，未涉及挠点（torsion point）计算和阶验证（order validation）。这些操作在完整密钥交换中占约 20-30% 时间，排除素数 3 虽能加速挠点搜索约 20%，但总体改进有限。
4. **未集成最新优化**：Pope 等 2025 提出的基于配对的验证方法（声称 4 倍加速），本文未集成。未来工作可探索将其与 DACsHUND/无虚拟 Matryoshka 结合以进一步提升性能。

**未来方向**：
- **配对验证集成**：将 Pope 等的配对验证方法整合到本文框架中，可能将完整密钥交换加速至接近 dCTIDH 的水平。
- **自动化参数优化**：开发更高效的搜索算法（如遗传算法、模拟退火）以在 DACsHUND 约束下找到更优配置。
- **硬件加速**：探索 FPGA/ASIC 实现，利用并行化批处理减少常量时间约束下的性能开销。
- **形式化验证**：对 DACsHUND 和无虚拟 Matryoshka 的安全性进行形式化证明，确保在各种攻击模型下都无漏洞。

---

**总结**：本文首次实现了同时满足确定性、常量时间和完全无虚拟操作的 CSIDH 类协议高效实现，通过 DACsHUND 和无虚拟 Matryoshka 的创新设计，在安全性显著提升的前提下仅增加 12-14% 的性能开销，为后量子密码学的实际部署提供了一个重要的安全-性能平衡点。
