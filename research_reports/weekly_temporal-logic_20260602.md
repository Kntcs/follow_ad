## 1. [2507.21598] A tree-shaped tableau for checking the satisfiability of signal temporal logic with bounded temporal operators

### 核心内容

本文提出了一种新颖的树形、单遍tableau方法，用于检查离散时间STL（Signal Temporal Logic，信号时序逻辑）的可满足性问题。该方法最初设计用于验证CPS（Cyber-Physical Systems，信息物理系统）需求集的一致性，但应用范围广泛，包括：需求一致性检查、生成满足需求的示例信号、验证STL公式的等价性和蕴含关系。

**关键创新**：与传统SMT编码方法不同，该tableau方法利用STL公式中大时间区间产生的冗余性来加速可满足性检查。核心思想是：在有限时间范围内验证需求一致性时，不需要检查每个时刻——对于时间域不相交的需求，它们天然一致；对于时间域相交的需求，只需在新约束施加的时刻进行检查，中间的时刻可以跳过。

**技术路线**：
- 将STL公式转换为严格范式（strict normal form）
- 通过扩展规则（expansion rules）构建tableau树
- 引入**JUMP规则**：当连续节点中时序公式的命题部分相同时，直接跳过冗余的中间时刻
- 使用SMT求解器检查线性约束的一致性

### 创新点

1. **时间跳跃优化（Temporal Jump）**：这是本文最核心的贡献。传统方法需要为每个时间步骤创建变量和约束，导致变量数量随时间区间长度线性增长。本文的JUMP规则通过识别重复节点模式，直接跳转到下一个关键时间点，显著减少了tableau树的节点数量。

2. **单遍树形tableau**：借鉴Reynolds的LTL tableau，设计了针对STL的单遍构造方法。与传统的图形两遍tableau（先构建图，再遍历寻找模型）不同，该方法在构建树的同时判断分支是否可接受。

3. **适用于长时间范围场景**：特别适合需要长期满足的需求验证（如飞机飞行时间），这是SMT方法的主要瓶颈所在。

4. **完备性和正确性证明**：论文提供了形式化的正确性（soundness）和完备性（completeness）证明，保证方法的理论可靠性。

5. **多种启发式优化**：论文第5节提出了多种启发式和优化策略，利用有界时序算子（如Globally和Finally）的分解性质提升效率。

### 技术细节

#### STL语法与语义
- **语法**：`φ ::= ⊤ | f(R)=k | f(R)>k | ¬φ | φ₁∨φ₂ | φ₁U_Iφ₂`
  - R ⊆ S：信号变量集合
  - f：线性函数
  - I = [a,b]：时间区间
- **派生算子**：Finally (F_I), Globally (G_I), Release (R_I), Strict Until (sU_I), Strict Release (sR_I)

#### 严格范式转换
任何公式可通过以下替换递归转换为严格范式（仅包含∨, ∧, sU, sR正形式）：
```
F_I φ ≡ ⊤ sU_I φ
G_I φ ≡ ¬F_I ¬φ
φ₁ U[a,b] φ₂ ≡ G[0,a] φ₁ ∧ φ₁ sU[a,b] (φ₁ ∧ φ₂)
```

#### Tableau构造规则

**扩展规则（Expansion Rules）**：
- **P类规则**（命题逻辑）：
  - `φ₁∨φ₂` → 分支为 {φ₁} 和 {φ₂}
  - `φ₁∧φ₂` → 单分支 {φ₁, φ₂}
- **S类规则**（时序算子）：
  - `F_I φ` 在 t∈I 时 → 分支为 {φ} 和 {F_I φ}（标记）
  - `G_I φ` 在 t∈I 时 → 单分支 {φ, G_I φ}（标记）

**STEP规则**：
从处于"准备状态"（poised）的节点u，创建子节点u'：
- `t(u') = t(u) + 1`
- 保留标记的时序算子
- 移除已耗尽区间的算子

**JUMP规则**（关键优化）：
当检测到节点重复模式时，选择最小的时间界 k > t(u)，直接跳转到时间k，避免逐步构建中间节点。

**终止规则**：
- **LOCALLY-UNSAT**：当约束集 {f(R)=k, f(R)>k} 不一致时拒绝
- **UNTIL**：当 sU 算子在区间耗尽时仍未满足则拒绝
- **EMPTY**：当节点不含时序算子且无拒绝条件则接受

#### 复杂度分析
论文证明了算法的时间复杂度，并通过实验验证在多数情况下优于SMT和FOL编码方法，特别是在大时间区间场景下优势明显。

## 2. [2603.14692 v1] Applications of Intuitionistic Temporal Logic to Temporal Answer Set Programming

### 核心内容

本文将直觉主义时序逻辑（Intuitionistic Temporal Logic）的理论基础扩展到时序回答集编程（Temporal Answer Set Programming），建立了时序直觉逻辑与时序逻辑编程之间的形式化对应关系。论文通过时序平衡逻辑（Temporal Equilibrium Logic, TEL）这一框架，将Pearce的平衡逻辑和Osorio的安全信念（safe beliefs）两种经典的不动点刻画方法提升到时序设置下，为时序回答集编程提供了更深层的逻辑理论基础。

**主要贡献：**
1. 将Pearce的理论完备化（theory completions）方法扩展到时序平衡逻辑
2. 将Osorio的安全信念方法从直觉逻辑扩展到时序直觉逻辑
3. 证明了在中间时序逻辑（intermediate temporal logic）下，安全信念集合保持不变
4. 建立了THT-时序安全信念与时序平衡模型的一一对应关系

### 创新点

**1. 语义重构方法**

论文采用纯语义方法替代了Osorio原有的句法方法，因为：
- 直觉时序逻辑缺乏完备的公理系统
- 命题变量的真值在时间上动态变化，无法直接应用句法转换
- 通过双模拟（bisimulation）技术实现了语义层面的证明

**2. 时序直觉逻辑的层次体系**

识别出保持一致性的"最弱"直觉时序逻辑ITL_BD_n作为基础：
- ITL_e（一般直觉时序逻辑）的一致性无法归约到LTL
- ITL_p（持久直觉时序逻辑）仍然不满足一致性归约
- ITL_BD_n（有限直觉深度）满足一致性等价：Γ is ITL_BD_n-consistent ⟺ Γ is LTL-consistent

**3. 技术突破**

通过两个关键引理实现模型收缩：
- **Lemma 10**：将满足□(¬p ∨ ¬¬p)的ITL_BD_n模型收缩为每个时间点只有一个极大世界的模型
- **Lemma 11（收缩引理）**：在满足特定条件下，可将ITL_BD_n模型收缩为THT模型，同时保持双模拟关系

### 技术细节

**1. 核心形式化定义**

**直觉时序框架（Intuitionistic Temporal Frame）：**
```
D = (W, ≼, S)
其中：
- W: Kripke世界集合
- ≼: 偏序关系（直觉蕴含）
- S: W → W 后继函数（时间流）
- 前向合流性：w ≼ v ⟹ S(w) ≼ S(v)
- 后向合流性：S(w) = v ∧ v ≼ u ⟹ ∃t(w ≼ t ∧ S(t) = u)
```

**时序安全信念定义：**
```
T ⊆ P^◦ 是Γ的ITL_BD_n-时序安全信念 ⟺

1) Γ ∪ {◦^i¬¬p | ◦^i p ∈ T} ∪ {◦^i¬p | ◦^i p ∉ T} 是ITL_BD_n-一致的

2) Γ ∪ {◦^i¬¬p | ◦^i p ∈ T} ∪ {◦^i¬p | ◦^i p ∉ T} ⊨_ITL_BD_n T

其中 P^◦ = {◦^i p | p ∈ P, i ≥ 0} 为时序原子集
```

**2. 双模拟技术（Bisimulation for ITL）**

直觉时序双模拟关系Z ⊆ W₁ × W₂需满足9个条件：
- **C1**: 原子命题一致性 w₁Zw₂ ⟹ V₁(w₁) = V₂(w₂)
- **C2-C3**: ≼的前向/后向模拟
- **C5**: 时间后继保持 w₁Zw₂ ⟹ S(w₁)ZS(w₂)
- **C6-C9**: Until/Release算子的四向模拟条件

**关键性质（Lemma 9）：**
```
若 w₁Zw₂，则对所有时序公式φ ∈ L_t：
M₁,w₁ ⊨ φ ⟺ M₂,w₂ ⊨ φ
```

**3. 主定理证明框架**

**Theorem 3（逻辑独立性）：**
```
对任何中间时序逻辑 ITL_BD_n ⊆ X ⊆ THT，
Γ的X-时序安全信念集合完全相同
```

**证明策略（Lemma 14的核心）：**
1. 假设T是Y-安全信念（Y较强）
2. 取任意ITL_BD_n模型M,w满足Γ ∪ {◦^i¬¬p|...} ∪ {◦^i¬p|...}
3. 应用Lemma 10：收缩为每时间点单极大世界的模型M'
4. 对每个时间点S^i(w')的子框架应用Lemma 11：收缩为THT模型M''
5. 利用双模拟传递性：M,w ⊨ T ⟸ M',w' ⊨ T ⟸ M'',（0,0）⊨ T
6. 由于T是THT-安全信念，推导出M'',（0,0）⊨ T
7. 反向传递得到M,w ⊨ T，从而证明T是ITL_BD_n-安全信念

**4. 时序平衡模型定义**

```
THT模型 M = ⟨(ℕ×{0,1}, ≼, S), V⟩ 是Γ的平衡模型 ⟺

1) M是完全的：V((i,0)) = V((i,1)) 对所有 i ≥ 0
2) M,(0,0) ⊨ Γ
3) 不存在M' < M使得M',(0,0) ⊨ Γ

其中 M' ≤ M 定义为：
- V'((i,1)) = V((i,1)) 对所有 i ≥ 0
- V'((i,0)) ⊆ V((i,0)) 对所有 i ≥ 0
```

**5. 一致性归约的关键**

**Lemma 8证明核心步骤：**
```
ITL_BD_n一致 ⟹ LTL一致：
给定 M,w ⊨ Γ 且 depth((W,≼),w) ≤ n
→ 存在极大世界 v 使得 w ≼ v
→ 构造LTL模型 M' = ⟨{S^n(v)|n≥0}, =, S'⟩
→ M',v ⊨ Γ（利用单调性）

LTL一致 ⟹ ITL_BD_n一致：显然（LTL是ITL_BD_1）
```

这种一致性归约保证了判定问题的可处理性，是将时序ASP实现为实用系统的理论基础。

## 3. [IJCAI 2025] Coalition obstruction temporal logic: A new obstruction logic to reason about demon coalitions

### 核心内容

Coalition Obstruction Temporal Logic (COTL) 是一种新型的时序逻辑框架，用于分析动态博弈场景中的防御者联盟（demon coalitions）。该逻辑扩展了 Obstruction Logic (OL)，核心思想是在博弈模型中，防御者（demons）可以通过停用图结构中的边来主动阻碍攻击者达成目标。

**研究动机：** 传统战略能力分析逻辑（如 ATL）将博弈模型视为静态实体——玩家的动作决定其在竞技场（arena）中的位置，但不会修改竞技场的结构。COTL 突破了这一限制，引入了动态博弈模型，允许防御者联盟通过策略性地禁用特定转移来改变系统结构。

**形式化框架：**
- **模型结构：** 由有向图、动作集合和代价函数组成，代价函数为每个状态下的每个动作分配成本
- **路径语义：** 不假设路径必须是无限的，允许有限路径和无限路径，使语义更接近 ATLf 等形式化方法
- **预算约束：** 防御者联盟（legions）受代价约束，表示为 Cost(last(h'), G, S(h')) ≤ n
- **边停用机制：** 在每个状态，动作组合（每个 demon 一个）可以临时禁用该状态的相邻边集合

### 创新点

1. **多防御者协作建模：** 首次在 Obstruction Logic 框架下形式化防御者联盟的协作阻碍策略，填补了单一防御者模型的空白

2. **资源受限的战略推理：** 引入基于代价的策略，每个动作都有成本，防御者必须在预算约束下做出最优阻碍决策

3. **动态结构操纵：** 与 ATL 等静态博弈逻辑不同，COTL 允许防御者通过停用边来动态改变博弈图的拓扑结构

4. **计算复杂度突破：** 模型检测问题的复杂度为 **PTIME-Complete**，证明可从 CTL 模型检测问题归约（已知为 PTIME-hard）

5. **安全博弈应用：** 特别适用于网络安全场景，其中防御者必须协作阻止攻击者目标，能够表达依赖于动作序列和防御者对博弈结构战略操纵的复杂时序属性

### 技术细节

**与相关工作的区别：**
- **vs. ATL（Alternating-time Temporal Logic）：** ATL 扩展 CTL 到多玩家场景，但博弈模型静态；COTL 引入动态边停用机制
- **vs. OL（Obstruction Logic）：** OL 关注单一防御者；COTL 扩展到防御者联盟并引入预算约束
- **vs. OATL（Obstruction Alternating-time Temporal Logic）：** OATL 结合了阻碍和交替时间推理，任何 ATL 公式可转换为 OATL 公式；COTL 聚焦于多防御者协作的代价约束策略
- **vs. TOL（Timed Obstruction Logic）：** TOL 在 OL 基础上加入实时元素和边代价；COTL 专注于联盟协作而非时间维度

**模型检测算法：**
论文第 2 节定义了语法和语义，提供了验证动态对抗环境中复杂安全属性的模型检测算法。通过定理 4 和定理 5 建立了复杂度界限。

**应用场景：**
- 网络安全中的攻击者-防御者博弈建模
- 需要多个防御机制协作的安全协议验证
- 资源受限环境下的威胁响应策略优化

## 4. [2311.01414] A dynamic temporal logic for quality of service in choreographic models

### 核心内容

本文提出了一个基于编排模型（choreographic model）的分布式系统服务质量（QoS）表达与分析框架。核心思想是将QoS约束建模为局部组件状态的契约（contracts），然后通过聚合（aggregation）来分析系统整体的QoS属性。框架基于两个关键模型：g-choreographies（全局视图）和Communicating Finite State Machines（CFSMs，局部视图），并引入了动态时序逻辑QL来表达和验证QoS属性。

### 创新点

1. **QoS扩展的CFSMs**：为CFSMs的每个状态关联QoS规范，使用Real-Closed Fields（RCFs，实闭域）抽象表示QoS值，确保可判定性
2. **动态时序逻辑QL**：受DLTL启发，用全局编排（g-choreographies）索引时序模态算子，实现对异步消息传递系统的QoS属性谓词描述
3. **半可判定性证明**：提供k-bounded半决策过程，支持有界模型检查方法验证QoS属性

### 技术细节

**1. QoS规范建模（Definition 3）**
- 扩展RCF理论表示：`Σ = {0,1} ∪ Q, {+,·} ∪ {⊕_a}_{a∈Q}, {<}`
- Q为QoS属性常量集（如时间t、成本c、内存m）
- 每个属性关联聚合算子⊕_a（如max用于内存、+用于成本）
- 约束公式Γ_1定义属性取值范围

**2. QoS扩展的CFSM（Definition 4）**
```
M_QoS = ⟨M, F, qos⟩
- M = ⟨Q, q_0, →⟩ 标准CFSM
- F ⊆ Q 终止状态集
- qos: Q → C 状态到QoS规范的映射
```

**3. 聚合函数（Example 9）**
对于运行π，聚合函数agg_S(π)计算累积QoS：
- f(π)：收集所有参与者局部状态的QoS规范
- g(π)：使用聚合算子计算属性累积值
  - 初始：`a = ⊕_{A∈P} a^{q_0(A)}_A`
  - 递归：`a = (⊕_{i<n,A=sbj(ℓ_i)} a^{q_i(A)}_A) ⊕_a (⊕_{A∈P} a^{q_n(A)}_A)`

**4. QL逻辑语法（Definition 5）**
```
Φ ::= ⊤ | ψ | ¬Φ | Φ ∨ Φ | Φ U_G Φ
```
- ψ：RCF理论中的原子公式（约束QoS属性）
- U_G：用全局编排G索引的until算子
- 模态算子：⟨G⟩Φ ≡ ⊤ U_G Φ, [G]Φ ≡ ¬⟨G⟩¬Φ

**5. QL语义（Definition 6）**
```
⟨π, π'⟩ ⊨_S Φ_1 U_G Φ_2 当且仅当
存在π_2使得：
  - L[π_2] ∈ L̂[G]（π_2是G的完整执行）
  - π'π_2 ∈ prf(π)（前缀条件）
  - ⟨π, π'π_2⟩ ⊨_S Φ_2
  - 对所有π_3 ∈ prf(π_2)，若π_3 ≠ π_2则⟨π, π'π_3⟩ ⊨_S Φ_1
```

**6. 半决策过程（Section 6）**
- qSat：枚举长度≤k的所有运行，检查是否满足公式
- qModels：递归检查公式结构，调用RCF决策过程验证原子公式
- qUntil：处理U算子，沿运行搜索见证
- Theorem 2：对于不含*算子的QL⁻片段，存在有限模型性质

**关键技术权衡**：
- 选择RCF而非quantales/c-semirings：确保可判定性，可用SMT求解器
- 聚合算子依赖属性语义（max用于内存峰值、sum用于累积成本）
- k-bounded检查：将完整QL转为QL⁻（展开*算子）进行有界验证

## 5. [FormaliSE 2025] Temporal logics meet real-world software requirements: a reality check

### 核心内容

本文通过 SpaceWire（航天通信协议标准）案例研究，系统性分析了时序逻辑理论能力与实际软件需求规约之间的表达能力鸿沟。研究团队从 SpaceWire 规范中提取 89 个具有时序行为的软件需求，使用不同时序逻辑（INV、LTL、CTL、MTLb 等）进行形式化建模，评估各逻辑框架的实用性与表达充分性。

**主要发现：**

## 9. [2502.20193] Model checking linear temporal logic with standpoint modalities

**主要贡献**：
1. 引入四种新的 SLTL 语义（原有 step 语义基础上增加 pobs、public、decr、incr）
2. 提供通用模型检验算法（基于历史自动机 history-DFA 构造）
3. 复杂度分析：step/public 语义 PSPACE 完全；pobs/decr/incr 在交替深度 1 时 PSPACE 完全；decr 有 EXPTIME 上界；incr 有 (|Ag|-1)-EXPSPACE 上界

### 创新点

1. **立场语义的系统化扩展**：首次系统性地定义五种 SLTL 语义，涵盖不同信息共享场景
   - **step 语义**：agent 仅知道经过的步数
   - **pobs 语义**（纯观察）：agent 可访问历史中属于其观察集 Pa 的原子命题真值（类似 LTLK 完美记忆语义）

   - **decr 语义**（递减）：嵌套立场公式 `⟨⟨b⟩⟩ψ` 在 `⟨⟨a⟩⟩φ` 中时，a 只能访问 Pa∩Pb 来推测 b 的知识
   - **incr 语义**（递增）：类似 decr，但 a 可访问 Pa∪Pb（联盟知识）

2. **更优的复杂度界**：模型检验比可满足性问题复杂度低
   - 可满足性：EXPSPACE 完全（已知结果）
   - 模型检验：step/public 语义下 PSPACE 完全（指数级降低）
   - 技术原因：模型检验可利用系统结构，历史自动机构造比一般可满足性检验更精细

3. **通用算法框架**：基于历史-DFA 的统一构造方法
   - 核心思想：通过产品构造和幂集构造生成历史自动机
   - O-确定性优化：利用观察集的确定性减少状态空间（Remark 5.1）
   - 递归构造：对嵌套立场公式的处理通过产品自动机实现

### 技术细节

#### 问题形式化

**输入**：
- SLTL 结构 T = (T₀, (Ta)a∈Ag, P, (Pa)a∈Ag)
  - T₀：主转换系统，建模实际系统
  - Ta：agent a 的转换系统，建模 a 立场下一致的执行
  - P：原子命题全集，Pa ⊆ P 为 a 的观察集
- SLTL 公式 φ
- 语义类型 * ∈ {step, pobs, public, decr, incr}

**输出**：判定 T ⊨* φ 是否成立（所有执行是否满足 φ）

**语义关键定义**（以 pobs 为例）：
```
T, h ⊨pobs ⟨⟨a⟩⟩φ  ⟺  
  ∃s ∈ Reach(Ta, h|Pa) ∃π ∈ Paths(Ta, s) ∃ρ ∈ (2^P)^ω :
    ρ|Pa = trace(π) ∧ ρ ⊨LTL φ
```
- `h|Pa`：历史 h 在 Pa 上的投影
- `Reach(Ta, h|Pa)`：Ta 中通过与 h|Pa 匹配的路径可达的状态集
- agent a 在观察 h|Pa 后，需存在一条路径 π，其扩展 ρ 满足 φ

#### 算法流程

**Phase 1: 历史-DFA 构造**（自底向上）
```
1. 对 LTL 子公式：构造标准 Büchi 自动机
2. 对立场子公式 χ = ⟨⟨a⟩⟩φ：
   a) 递归获得 φ 中所有立场子公式 χ₁,...,χₖ 的历史-DFA D₁,...,Dₖ
   b) 构造产品系统 T'χ = Ta^R ⊲⊳ D₁ ⊲⊳ ... ⊲⊳ Dₖ
      (Ta^R 是 Ta 的"丰富版本"，标签包含 P 中所有可能真值组合)
   c) 计算满足集 Uχ = Sat_T'χ(∃φ')  (可通过 Büchi 空性检验)
   d) 幂集构造 Dχ = pow(T'χ, Oa, Uχ)
      - 状态：Ta^R ⊲⊳ D₁ ⊲⊳ ... ⊲⊳ Dₖ 的状态子集
      - 转移：根据 Oa（观察集）确定性更新
      - 接受态：与 Uχ 相交非空的状态子集
```

**Phase 2: 模型检验**
```
构造产品自动机 T₀ ⊗ Dφ，检查 L(Dφ) ⊇ Traces(T₀)
```

**O-确定性优化**（关键技术）：
若 Di 是 Oχ-确定性的（观察历史唯一决定状态），则可将
```
pow(Ta^R ⊲⊳ D₁ ⊲⊳ ... ⊲⊳ Dₖ, Oχ)
```
重构为
```
pow(Ta^R ⊲⊳ ∏_{j≠i} Dj, Oχ) ⊲⊳ Di
```
状态数从 2^(|Ta| · ∏|Dj|) 降为 2^(|Ta| · ∏_{j≠i}|Dj|) · |Di|

#### 复杂度分析

**定理 5.6**：step 和 public 语义下，SLTL 模型检验问题可多项式归约到 LTLK₁ 模型检验，因此 PSPACE 完全。

**核心机制**：
- step 语义：历史-DFA 大小单指数界（Lemma 5.3）
- public 语义：类似，因所有 agent 共享完整历史
- pobs 语义（ad=1）：on-the-fly 构造避免显式幂集构造，PSPACE
- decr 语义：EXPTIME（利用观察集交集的单调性）
- incr 语义：(|Ag|-1)-EXPSPACE（观察集并集导致指数级增长）

**与可满足性的对比**：
- 可满足性：需考虑所有可能的转换系统，EXPSPACE 完全
- 模型检验：给定系统结构，可利用 O-确定性和最小化减小自动机尺寸

#### 实现细节

**历史自动机 vs LTLK**：
- LTLK 标准算法（如 [5]）：逐层幂集构造，每层指数膨胀
- 本文算法：细粒度分析，利用 SLTL 结构（观察集、立场系统分离）
- 关键差异：检测 O-确定性子公式，避免冗余幂集步骤

**空间优化**（Section 5）：
对交替深度 d > 1，使用 (d-1)-EXPSPACE 算法：
1. 识别深度 d-1 的极大立场子公式 Ψ
2. 为 Ψ 中公式构造历史-DFA（(d-1)-指数大小）
3. 引入新命题 pψ 替换 ψ ∈ Ψ，得到深度 1 公式 φ'
4. 在修改后的系统 T' 上运行多项式空间算法
