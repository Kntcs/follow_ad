| OpenVLA | 48.7% | 81.2% |
| π0 | 56.5% | 85.3% |
| PerAct（点云方法） | 58.9% | 87.1% |

**关键发现**：
- 零样本性能超越所有基线（包括专门为3D设计的PerAct）
- 10-shot微调后达到89.5%，证明网格重离散化的有效性

**3. 真实Franka机器人（空间推理任务）**

| 任务 | SpatialVLA | OpenVLA | 差距 |
|------|-----------|---------|------|
| Place on Shelf（放到架子指定层） | 88% | 62% | +26% |
| Stack Pyramid（堆金字塔） | 82% | 58% | +24% |
| Insert Peg（插销孔对齐） | 91% | 71% | +20% |

**分析**：需要精确深度判断的任务上，SpatialVLA优势显著

### 消融实验

**实验A：Ego3D编码的贡献**

| 输入模态 | LIBERO-Spatial | RLBench |
|---------|---------------|---------|
| 仅RGB | 82.3% | 51.2% |
| RGB+深度图（直接叠加） | 85.7% (+3.4%) | 54.8% |
| RGB+Ego3D（本文） | 95.2% (+12.9%) | 62.3% |

**结论**：Ego3D编码比直接用深度图高9.5%，证明显式3D坐标的重要性

**实验B：自适应网格 vs 固定bins**

| 离散化方法 | Token数/维度 | 成功率 | 推理速度 |
|-----------|------------|--------|---------|
| 固定256 bins | 1 | 88.2% | 29 Hz |
| 固定512 bins | 1 | 89.1% | 24 Hz |
| 3级网格（本文） | 3 | 88.9% | 110 Hz |

**分析**：
- 网格方法用3个token达到512 bins的精度
- 推理速度快4.6倍（因为并行预测3个token，而自回归预测512个分类头更慢）

**实验C：网格层级数的影响**

| 层级 | 等效分辨率 | Token数 | 成功率 |
|------|-----------|---------|--------|
| 2级（64×64） | 4096 | 2 | 86.5% |
| 3级（32×32×32） | 32768 | 3 | 88.9% |
| 4级（16×16×16×16） | 65536 | 4 | 89.2% |

**结论**：3级是性价比最优（4级仅高0.3%但token数增加33%）

**个人点评**:
这篇工作的亮点是**用显式3D先验增强VLA**，而非单纯靠大模型隐式学习。Ego3D编码设计优雅且计算高效，自适应网格解决了跨机器人迁移的痛点。局限性：(1) 依赖深度传感器（RGB-D相机或立体视觉），纯RGB场景无法应用；(2) 手眼标定误差会影响3D坐标精度；(3) 网格重离散化仍需100个演示，对快速适配新机器人仍有门槛。未来方向：探索用单目深度估计替代深度传感器，降低硬件依赖。

## 9. [2506.01844] SmolVLA: Overcoming Data Limitations in Robotics using Small Vision-Language-Action Models

**一句话总结**: 用450M参数+社区开源数据（仅2.3万episode），通过层跳跃和异步推理实现87.3%成功率和2倍吞吐

**背景与动机**:
VLA领域存在"数据垄断"问题：OpenVLA用97万条轨迹，π0用100万+条，但这些数据多数未公开，小团队和研究者只能用Open X-Embodiment等开源数据集（约2.3万episode）。如何在有限数据下训练强VLA模型成为关键挑战。

**核心创新**:

1. **层跳跃（Layer Skipping）**：砍掉预训练VLM的后半层（如PaliGemma的12层只保留前6层），原因是前N层的视觉-语言特征已足够用于动作预测，后半层的高层语义（如生成完整句子）对机器人控制冗余。

2. **交错注意力（Interleaved Attention）**：在保留的transformer层中，交替使用交叉注意力（CA，融合视觉-语言）和自注意力（SA，轨迹内部平滑），平衡信息提取和动作连贯性。

3. **异步推理栈（Asynchronous Inference）**：解耦动作预测和执行，在机器人执行当前动作时，后台异步预测下一个action chunk，吞吐提升2倍。

4. **社区数据标准化**：用VLM自动标注任务描述，人工映射相机视角，将不同来源数据统一到标准格式。

**技术方案**:

### 整体架构

```
输入：图像 + 语言 + 状态
↓
预训练VLM前6层（砍掉后6层）+ LoRA微调
  ├─ 层1-3：交叉注意力（CA）提取多模态特征
  ├─ 层4-6：自注意力（SA）平滑轨迹
↓
扩散策略头（10步去噪）
↓
输出：连续动作chunk（H=16）
```

### 关键技术点

**1. 层跳跃的理论依据**

**实验验证**（在LIBERO上测试不同层数）：
```
保留12层（完整）：87.3%成功率，推理50ms
保留6层：87.3%（性能无损），推理28ms
保留3层：82.1%（性能下降），推理15ms
```

**为什么前6层足够**：
- 预训练VLM的前N层学习视觉基础特征（边缘、纹理、物体部件）
- 后半层学习语言生成能力（语法、语义连贯性），但机器人只需理解指令，不需生成文本
- 论文用CKA（Centered Kernel Alignment）分析特征相似度，发现第6层特征与第12层相关性>0.95

**2. 交错注意力的设计**

**传统transformer**（如OpenVLA）：
```
每层都包含CA（交叉注意力）+ SA（自注意力）+ FFN
```

**SmolVLA的交错设计**：
```
层1：CA（融合图像-文本） + FFN
层2：SA（时序平滑） + FFN
层3：CA + FFN
层4：SA + FFN
层5：CA + FFN
层6：SA + FFN
```

**为什么这样设计**：
- CA负责信息提取（从图像和指令中抽取任务相关特征）
- SA负责轨迹平滑（让相邻时刻的动作连贯）
- 交替使用避免过度融合导致的信息干扰

**数学形式**：
- CA层：Attn(Q=action_tokens, K=V=concat(img_tokens, text_tokens))
- SA层：Attn(Q=K=V=action_tokens)

**消融实验**：
- 全CA：85.2%（动作抖动）
- 全SA：81.7%（无法理解新指令）
- 交错CA/SA：87.3%（最优）

**3. 异步推理栈**

**同步推理（传统方法）**：
```
t=0: 预测动作chunk A1 (50ms) → 执行A1 (160ms)
t=210ms: 预测A2 (50ms) → 执行A2 (160ms)
总周期：210ms，吞吐4.8Hz
```

**异步推理（SmolVLA）**：
```
t=0: 预测A1 (50ms) → 执行A1开始
t=50ms: 后台预测A2（与A1执行并行）
t=160ms: A1执行完，A2已ready，立即执行A2
总周期：160ms，吞吐6.25Hz
```

**实现细节**：
- 用两个GPU或单GPU双stream
- 设置队列阈值：当剩余执行时间<预测时间时触发预测
- 公式：`if remaining_exec_time < E[prediction_time]: start_async_predict()`

**4. 社区数据标准化**

**问题**：Open X-Embodiment包含40+数据集，格式不统一
- 相机视角不同（wrist/third-person/overhead）
- 任务描述不一致（有些无标注）
- 动作空间不同（关节/笛卡尔/增量）

**SmolVLA的处理**：
- 用GPT-4V自动标注缺失的任务描述
- 手动映射相机视角（建立视角→标准化位置的映射表）
- 统一动作归一化（所有维度映射到[-1,1]）

**实验效果**:

**主要结果**：

**1. LIBERO基准**：
- SmolVLA-0.45B：87.3%（仅用2.3万episode）
- π0-3B：86.0%（用100万+episode）
- OpenVLA-7B：76.5%

**2. Meta-World**：
- SmolVLA：Very Hard任务60%
- π0：44%

**3. 真实机器人SO-100**：
- SmolVLA：78.3%
- π0（7倍参数）：81.2%

**消融实验**：
- 完整SmolVLA：87.3%
- 去掉层跳跃（用12层）：87.3%，速度慢1.8倍
- 去掉交错注意力：85.2%
- 去掉异步推理：87.3%，吞吐减半

**个人点评**:
这篇工作证明**数据效率比模型规模更重要**——用45倍少的数据、7倍小的模型达到更好性能。层跳跃和交错注意力是实用的工程优化，异步推理栈解决了实际部署痛点。开源全部代码和训练配方，降低了VLA研究门槛。

## 10. [2502.05855] DexVLA: Vision-language model with plug-in diffusion expert for general robot control

**一句话总结**: 用10亿参数扩散专家+三阶段课程学习，仅需100小时数据实现灵巧操作（折叠衣服0.92分，基线完全失败）

**背景与动机**:
现有VLA只扩大视觉-语言模块（7-70B），但动作专家仅数百万参数，导致"看得懂但手不够灵"。灵巧操作（如折叠衣服、倒水）需要复杂的运动技能，小动作专家容量不足。

**核心创新**:

1. **10亿参数扩散专家**：将动作生成模块从100M扩大到1B，用多头架构支持跨机器人学习
2. **三阶段课程学习**：Stage 1跨机器人预训练 → Stage 2具体机器人对齐 → Stage 3任务专精
3. **内置高层规划**：通过子步骤推理（substep reasoning）实现端到端规划，无需外部规划器

**技术方案**:

**整体架构**：
- 基座：Qwen2-VL（7B）+ 10亿参数扩散专家
- 训练：三阶段渐进式学习
- 推理：60Hz实时控制（单A6000 GPU）

**关键技术**：

**1. 扩散专家设计**：
- 基于ScaleDP的Transformer变体
- 多头输出（每种机器人一个头）
- 去噪过程：ε_θ(a_t, o, t) 预测噪声

**2. 三阶段训练**：
- Stage 1：只训练扩散专家，学习基础运动技能
- Stage 2：连接VLM和扩散专家，在特定机器人上对齐
- Stage 3：针对复杂任务（如折叠）精调，引入子步骤推理

**3. 子步骤推理标注**：
- 每5秒标注一条子指令
- 模型自己生成这些子指令（推理Token）
- 通过FiLM层调节扩散专家

**实验效果**:

**主要结果**：

**1. 无任务专精（Stage 2后）**：
- 折叠衣服：DexVLA 0.92 vs 基线全部0.0
- 垃圾分拣：0.77 vs OpenVLA 0.15
- 清理桌子：0.72 vs OpenVLA 0.3

**2. 新机器人快速学习（100个演示）**：
- 灵巧手倒水：0.85 vs 基线全部<0.1
- 双臂打包：0.95 vs DP 0.35

**3. 超长任务（Stage 3）**：
- 洗衣折叠（2分钟+）：0.4 vs π0 0.2
- 复杂清理：0.7 vs π0 0.63

**消融实验**：
- 无Stage 1预训练：完全失败
- 扩散专家93M→410M→1B：0.17→0.63→0.92

**个人点评**:
首次系统验证"动作专家也需要大模型"，三阶段课程学习优雅且数据高效（100小时 vs 4000小时）。内置推理能力消除外部依赖。局限：10亿参数推理成本较高，未探索5B+规模的性能上限。
